## Обзор

Rox не назначает диапазоны портов для каждого рабочего пространства. Он обнаруживает прослушиваемые порты от процессов, запущенных в каждом рабочем пространстве, и позволяет управлять ими из интерфейса.

![Port Panel](/images/ports.png)

## Возможности

- **View active ports** — просмотр того, какие процессы используют какие порты
- **Kill processes** — остановка процесса кликом по его порту
- **Workspace grouping** — порты группируются по рабочему пространству, которому принадлежит процесс
- **Terminal focus** — выберите порт, чтобы вернуться к терминалу, которому он принадлежит
- **Browser actions** — открытие локальных веб-серверов во встроенном браузере или во внешнем браузере из действия порта

## Метки портов

Добавляйте понятные имена для автоматически обнаруженных портов с помощью
файла конфигурации рабочего пространства. Полезно для:

- Предоставления понятных меток для вашей команды
- Упрощения визуального поиска часто используемых портов dev-серверов в интерфейсе

Создайте `.rox/ports.json` в вашем репозитории:

```json
{
  "ports": [
    { "port": 3000, "label": "Frontend Dev Server" },
    { "port": 8080, "label": "API Server" },
    { "port": 5432, "label": "PostgreSQL" }
  ]
}
```

**Поля:**
- `port` — номер порта (1-65535)
- `label` — отображаемый текст, показываемый во всплывающей подсказке

**Поведение:**
- Динамическое обнаружение портов по-прежнему имеет приоритет
- `ports.json` лишь маркирует порты, которые Rox уже обнаруживает как прослушиваемые
- Порты без подходящей метки всё равно отображаются
- Записи меток для портов, которые сейчас не прослушиваются, игнорируются
- Каждое рабочее пространство читает метки из файла собственного worktree
- Изменения обнаруживаются автоматически
- Порты можно открыть по адресу `localhost:PORT` из действия браузера

**Обработка ошибок:**

Если `ports.json` повреждён:
- Метки из этого файла игнорируются до исправления
- Обнаруженные порты по-прежнему отображаются без этих меток

**Советы:**
- Закоммитьте `.rox/ports.json`, чтобы поделиться метками портов с вашей командой
- **Совет профи:** если вам нужны детерминированные диапазоны портов для каждого рабочего пространства, реализуйте это в скриптах setup/teardown, резервируя диапазон в общем файле (например, `~/.rox/port-allocations.json`) во время setup и освобождая его во время teardown. Смотрите примеры в этом репозитории: [`.rox/setup.sh`](https://github.com/agisota/rox/blob/main/.rox/setup.sh) и [`.rox/teardown.sh`](https://github.com/agisota/rox/blob/main/.rox/teardown.sh).

## Обнаружение и обновления

Обнаружение портов выполняется в каждом host-сервере, а не в десктопном рендерере. Host-сервер отслеживает деревья процессов терминала, сканирует прослушиваемые порты, сопоставляет любую подходящую метку порта и публикует события `port:changed`, когда порты появляются или исчезают.

Боковая панель десктопа поддерживает один запрос портов на каждый онлайн-хост. Она обновляет кешированный снимок хоста по событиям портов, группируя всплески, чтобы обновления оставались отзывчивыми без повторной выборки при каждом изменении порта. При этом более медленная резервная повторная выборка всё равно выполняется, чтобы интерфейс восстановился, если событие было пропущено при переподключении.
