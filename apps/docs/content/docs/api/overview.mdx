## Обзор

Публичный API Rox — это типобезопасный tRPC-роутер, смонтированный в бэкенде (`apps/api`). Один корневой роутер (`appRouter`) объединяет все домены; клиенты — веб-приложение, десктоп, мобильное приложение, CLI и [TypeScript SDK](/sdk/getting-started) — обращаются к нему через один и тот же контракт.

<Callout type="info">
Источник истины для поверхности API — `packages/trpc/src/root.ts`. Этот пакет (`@rox/trpc`) переиспользуется всеми клиентами, поэтому типы входов/выходов одинаковы на вебе, десктопе и в SDK.
</Callout>

## Транспорт

Запросы идут по двум маршрутам в зависимости от того, обслуживает их облако или конкретный хост:

| Маршрут | Базовый URL | Когда используется | Источник |
| --- | --- | --- | --- |
| **Облако** | `https://api.rox.one` (`ROX_BASE_URL`) | Облачные данные: задачи, проекты, организация, биллинг | `apps/api`, `packages/sdk/src/client.ts` |
| **Relay → хост** | `https://relay.rox.one/hosts/<routingKey>/trpc/<procedure>` (`ROX_RELAY_URL`) | Операции на хосте: рабочие пространства, терминалы, запуски агентов | `apps/relay`, `packages/sdk/src/client.ts` |

HTTP-эндпоинт tRPC монтируется в `apps/api/src/app/api/trpc/[trpc]/route.ts`. Операции, привязанные к машине разработчика, проксируются через [Rox Relay](/self-host/overview) к нужному хост-серверу — хост должен быть в сети.

## Аутентификация

- **Сессия** — клиенты входят через better-auth (см. [Модель аутентификации](/security/auth-model)). Веб/десктоп/мобайл используют cookie-сессии.
- **API-ключи** — серверные интеграции и SDK передают ключ `sk_live_…` (см. [API-ключи](/security/api-keys)). Ключи долгоживущие и дают полный доступ к организации — не передавайте их во фронтенд.
- **Контекст организации** — запросы выбирают активную организацию через заголовок `x-rox-organization-id`; SDK задаёт его из `organizationId`.

## Домены API

Корневой роутер регистрирует следующие домены (`packages/trpc/src/root.ts`). Это полная карта поверхности; публичный SDK отражает их подмножество (см. [Домены SDK](/api/sdk-domains)).

| Домен | Назначение |
| --- | --- |
| `achievements` | Достижения пользователя |
| `admin` | Админ-панель: пользователи, баланс, флаги, выручка |
| `agent` | Конфигурация и запуски агентов |
| `agentRole` | Роли агентов в пайплайнах |
| `agentSource` | Источники определений агентов |
| `apiKey` | Жизненный цикл API-ключей |
| `analytics` | Аналитика и метрики |
| `automation` | Регулярные автоматизации (RRULE) |
| `chat` | Чат и сессии чата |
| `device` | Регистрация устройств и device/CLI-аутентификация |
| `economy` | Баланс, лимиты, метрика потребления |
| `executionCircuit` | Контур исполнения workflow |
| `graph` | Граф зависимостей/связей |
| `host` | Хосты (машины разработчиков) |
| `identity` | Идентичности пользователя (handle, провайдеры) |
| `integration` | Внешние интеграции (Linear, GitHub, …) |
| `journal` | Журнал/память по проекту |
| `knowledge` | База знаний |
| `memory` | Долговременная память |
| `notes` | Заметки |
| `organization` | Организация и участники |
| `pipeline` | Пайплайны агентов |
| `pipelineTrigger` | Триггеры пайплайнов |
| `profile` | Профиль пользователя |
| `project` | Проекты (репозитории) |
| `ranking` | Ранжирование/скоринг |
| `runtime` | Рантайм-конфигурация |
| `share` | Шеринг и share-ссылки |
| `skill` | Навыки и библиотеки навыков |
| `support` | Поддержка |
| `task` | Задачи и статусы задач |
| `team` | Команды |
| `usage` | Учёт потребления |
| `user` | Пользователь и аккаунт |
| `v2Host` | Хосты (модель v2) |
| `v2Project` | Проекты (модель v2) |
| `v2Workspace` | Рабочие пространства (модель v2) |
| `voice` | Голосовые сессии |
| `workflow` | Workflow |
| `workspace` | Рабочие пространства (worktree) |

<Callout type="info">
Не все домены доступны публично. Многие обслуживают внутренние нужды клиентских приложений (десктоп, веб). Стабильная программная поверхность — это [TypeScript SDK](/sdk/getting-started), который отражает домены, безопасные для внешнего использования.
</Callout>
