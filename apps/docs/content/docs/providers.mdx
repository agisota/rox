Если вы используете Rox в терминале — любой CLI-агент (Claude Code, Codex, Curser Agents и т. д.)
автоматически просто работает. В этом и есть прелесть нашего подхода «терминал прежде всего».

Для интерфейса Chat у нас есть процесс, который помогает пользователям начать работу.

## Chat UI

1. Откройте выбор модели внизу чата

![Model picker](/images/providers-model-picker.png)

2. Нажмите на **иконку ключа** рядом с **Anthropic**

3. Выберите **Use API key**

![Connect Anthropic](/images/providers-connect-dialog.png)

4. Введите переменные окружения для вашего провайдера (по одной на строку, в формате `VAR_NAME=value`)

![Environment variables](/images/providers-env-vars.png)

5. Нажмите **Save settings**

См. разделы по провайдерам ниже, чтобы узнать конкретные переменные для использования.

## Terminal

Задайте переменные окружения в профиле вашей оболочки или для каждого рабочего пространства через **Settings > Env**.

<Callout type="info">
Для Anthropic-совместимых провайдеров (OpenRouter, Vercel AI Gateway) установите `ANTHROPIC_API_KEY` в пустую строку, чтобы предотвратить прямую аутентификацию Anthropic.
</Callout>

## Vercel AI Gateway

Маршрутизируйте запросы через [Vercel AI Gateway](https://vercel.com/docs/ai-gateway).

```bash
ANTHROPIC_BASE_URL=https://ai-gateway.vercel.sh
ANTHROPIC_AUTH_TOKEN=your-vercel-ai-gateway-api-key
ANTHROPIC_API_KEY=
```

## OpenRouter

[OpenRouter](https://openrouter.ai) предоставляет доступ ко многим моделям через единый API.

```bash
ANTHROPIC_BASE_URL=https://openrouter.ai/api
ANTHROPIC_AUTH_TOKEN=your-openrouter-api-key
ANTHROPIC_API_KEY=
```

## AWS Bedrock

Используйте модели Claude через ваш аккаунт AWS с помощью [Amazon Bedrock](https://aws.amazon.com/bedrock/).

Убедитесь, что ваши учётные данные AWS настроены через `aws configure`, переменные окружения или профиль SSO.

```bash
CLAUDE_CODE_USE_BEDROCK=1
AWS_REGION=us-east-1
```

При необходимости переопределите регион для маленькой/быстрой модели (Haiku):

```bash
ANTHROPIC_SMALL_FAST_MODEL_AWS_REGION=us-west-2
```

<Callout type="warn">
Закрепляйте конкретные версии моделей, чтобы предотвратить поломки при выпуске новых моделей:

```bash
ANTHROPIC_DEFAULT_OPUS_MODEL=us.anthropic.claude-opus-4-6-v1
ANTHROPIC_DEFAULT_SONNET_MODEL=us.anthropic.claude-sonnet-4-6
ANTHROPIC_DEFAULT_HAIKU_MODEL=us.anthropic.claude-haiku-4-5-20251001-v1:0
```
</Callout>

Полные сведения о настройке, включая конфигурацию IAM, см. в [документации Claude Code на Amazon Bedrock](https://code.claude.com/docs/en/amazon-bedrock).

## Google Vertex AI

Запускайте модели Claude на [Google Cloud Vertex AI](https://cloud.google.com/vertex-ai).

Аутентифицируйтесь с помощью `gcloud auth application-default login`, затем задайте:

```bash
CLAUDE_CODE_USE_VERTEX=1
CLOUD_ML_REGION=global
ANTHROPIC_VERTEX_PROJECT_ID=your-project-id
```

<Callout type="warn">
Закрепляйте конкретные версии моделей, чтобы предотвратить поломки при выпуске новых моделей:

```bash
ANTHROPIC_DEFAULT_OPUS_MODEL=claude-opus-4-6
ANTHROPIC_DEFAULT_SONNET_MODEL=claude-sonnet-4-6
ANTHROPIC_DEFAULT_HAIKU_MODEL=claude-haiku-4-5@20251001
```
</Callout>

Полные сведения о настройке, включая конфигурацию IAM, см. в [документации Claude Code на Google Vertex AI](https://code.claude.com/docs/en/google-vertex-ai).

## Microsoft Foundry (Azure)

Используйте модели Claude через [Microsoft Foundry](https://ai.azure.com).

Аутентифицируйтесь с помощью API-ключа или Microsoft Entra ID (`az login`), затем задайте:

```bash
CLAUDE_CODE_USE_FOUNDRY=1
ANTHROPIC_FOUNDRY_RESOURCE=your-resource-name

# If using API key authentication:
ANTHROPIC_FOUNDRY_API_KEY=your-azure-api-key
```

<Callout type="warn">
Закрепляйте конкретные версии моделей, чтобы предотвратить поломки при выпуске новых моделей:

```bash
ANTHROPIC_DEFAULT_OPUS_MODEL=claude-opus-4-6
ANTHROPIC_DEFAULT_SONNET_MODEL=claude-sonnet-4-6
ANTHROPIC_DEFAULT_HAIKU_MODEL=claude-haiku-4-5
```
</Callout>

Полные сведения о настройке, включая конфигурацию RBAC, см. в [документации Claude Code на Microsoft Foundry](https://code.claude.com/docs/en/microsoft-foundry).

## Другие Anthropic-совместимые провайдеры

Любой провайдер, предоставляющий Anthropic-совместимый API (например, LiteLLM), может быть использован:

```bash
ANTHROPIC_BASE_URL=https://your-provider.example.com
ANTHROPIC_AUTH_TOKEN=your-api-key
ANTHROPIC_API_KEY=
```

Сведения о конфигурации шлюза LLM см. в [документации Claude Code по шлюзам LLM](https://code.claude.com/docs/en/llm-gateway).
