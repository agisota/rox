---
title: Интеграция с Linear
description: Открывайте issues из Linear как рабочие пространства Rox с помощью пользовательского скрипта
---

Функция [пользовательского инструмента для написания кода](https://linear.app/docs/open-issues-with-custom-scripts)
в Linear может передать issue любой локальной команде. С помощью небольшого скрипта-обёртки
**Work on issue → Custom script** создаёт рабочее пространство Rox для issue —
с переключением на ветку issue, с уже работающим над ней агентом — и
открывает его в десктопном приложении.

## Как это работает

Linear здесь не использует схему URL. Он запускает локальный исполняемый файл, определённый в
`~/.linear/coding-tools.json`, и передаёт контекст issue через переменные окружения `LINEAR_*`
(`LINEAR_ISSUE_IDENTIFIER`, `LINEAR_ISSUE_BRANCH_NAME`,
`LINEAR_PROMPT` и другие). Скрипт-обёртка объединяет два вызова CLI:

1. `rox workspaces create` — создаёт локальное рабочее пространство на ветке
   issue и запускает агента с issue в качестве его промпта.
2. `rox workspaces open` — открывает это рабочее пространство в десктопном приложении через его
   deep link.

## Предварительные требования

- Установленный и авторизованный [Rox CLI](/docs/cli/getting-started)
  (`rox auth login`).
- Установленное десктопное приложение Rox (скрипт открывает рабочие пространства в нём).
- Проект, настроенный локально. Найдите его ID с помощью:

```bash
rox projects list
```

Скопируйте `id` проекта, для которого вы хотите открывать issues.

## 1. Создайте скрипт-обёртку

Сохраните это как `~/.linear/open-in-rox.sh` и замените `PROJECT` на ID
из предыдущего шага:

```bash
#!/usr/bin/env bash
# Linear custom script -> open the issue as a Rox workspace.
set -euo pipefail

PROJECT="<your-project-id>"

NAME="${LINEAR_ISSUE_IDENTIFIER:-linear-task}"
BRANCH="${LINEAR_ISSUE_BRANCH_NAME:-$NAME}"
PROMPT="${LINEAR_PROMPT:-Work on Linear issue ${NAME}.}"

# Make sure the local host service is running (idempotent).
rox start --daemon >/dev/null 2>&1 || true

result="$(
  rox workspaces create \
    --local \
    --project "$PROJECT" \
    --name "$NAME" \
    --branch "$BRANCH" \
    --agent claude \
    --prompt "$PROMPT" \
    --json
)"

id="$(printf '%s' "$result" | jq -r '.workspace.id')"

rox workspaces open "$id"
```

Сделайте его исполняемым:

```bash
chmod +x ~/.linear/open-in-rox.sh
```

<Callout type="info">
Скрипт зависит от наличия `rox` в вашем `PATH` и установленного `jq`.
Если `rox` отсутствует в вашем `PATH`, используйте его полный путь (десктопное приложение устанавливает
shim в `~/.rox/bin/rox`).
</Callout>

Ветка, названная по issue, ещё не будет существовать — `workspaces create` ответвляет её
от ветки проекта по умолчанию автоматически. Повторный запуск для того же issue
переиспользует существующее рабочее пространство вместо создания дубликата.

## 2. Укажите Linear на скрипт

Создайте `~/.linear/coding-tools.json`:

```json
{
  "openIssue": {
    "path": "/Users/you/.linear/open-in-rox.sh",
    "env": [
      "LINEAR_ISSUE_IDENTIFIER",
      "LINEAR_ISSUE_BRANCH_NAME",
      "LINEAR_PROMPT"
    ]
  }
}
```

Используйте абсолютный путь к вашему скрипту (`~` здесь не раскрывается).

<Callout type="info">
Если Linear перегенерирует этот файл со стартовой конфигурацией при включении
функции, верните блок `openIssue` обратно слиянием, а не позволяйте ему перезаписать
вашу версию.
</Callout>

## 3. Включите и запустите

1. В Linear откройте **Settings → Code & reviews → Configure coding tools** и
   включите **Custom script**.
2. На любом issue выберите **Work on issue → Custom script**. В первый раз
   Linear попросит вас **Choose a directory** — это рабочий каталог,
   передаваемый скрипту, поэтому выберите ваш репозиторий. (Приведённый выше скрипт его не читает,
   но Linear требует указать его.)

Linear запускает скрипт, и рабочее пространство открывается в Rox с агентом,
работающим над issue.

<Callout type="info">
В любом диалоге выбора файлов в macOS нажмите **⌘⇧.**, чтобы показать скрытые папки с точкой, или
**⌘⇧G**, чтобы ввести путь напрямую.
</Callout>

## Настройка

- **Без агента.** Чтобы открыть пустое рабочее пространство на ветке без запуска
  агента, уберите флаги `--agent` и `--prompt` (они требуются вместе).
- **Другой агент.** Замените `claude` на любой id пресета агента (`codex`,
  `opencode`, …).
- **Несколько репозиториев.** ID проекта зафиксирован в скрипте. Чтобы направлять разные
  команды или проекты Linear к разным проектам Rox, ветвитесь по
  переменной `LINEAR_*` (например, `LINEAR_PROJECT_NAME`) и устанавливайте `PROJECT`
  соответственно. Массив `env` в `coding-tools.json` — это allowlist: любая
  переменная `LINEAR_*`, которую вы читаете в скрипте, должна быть также добавлена туда, иначе она
  не будет присутствовать в окружении скрипта.
