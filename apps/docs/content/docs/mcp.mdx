Этот документ описывает **MCP v2**. Сервер v1 устарел, но по-прежнему доступен по адресу `https://api.rox.one/api/agent/mcp` для существующих интеграций.
</Callout>

## Обзор

Rox предоставляет [MCP-сервер (Model Context Protocol)](https://modelcontextprotocol.io), который позволяет AI-агентам программно взаимодействовать с вашими задачами, рабочими пространствами и устройствами.

## Возможности

| Категория | Операции |
|----------|------------|
| **Задачи** | список, получение (по id или slug), создание, обновление, удаление |
| **Рабочие пространства** | список (с фильтрацией по хосту), создание на хосте, удаление |
| **Автоматизации** | список, получение метаданных, чтение/запись тела промпта, создание, обновление метаданных, пауза, возобновление, запуск по требованию, список запусков, удаление |
| **Проекты** | список проектов, создание из Git URL или пути к локальному репозиторию |
| **Хосты** | список хостов (зарегистрированных машин), к которым у вас есть доступ |

### Ручная настройка

Кроме того, вы можете вручную настроить MCP-сервер для каждого клиента:

<Tabs items={["Claude Code", "Claude Desktop", "Cursor", "Codex", "Gemini CLI", "Open Code", "Amp CLI"]}>

  }

## Аутентификация

MCP-сервер использует OAuth 2.1. При первом подключении вам будет предложено авторизовать соединение в браузере. Токен ограничен областью вашей активной организации.

### Аутентификация по API-ключу

Для headless-окружений, CI/CD-конвейеров или скриптов, где аутентификация через браузерный OAuth непрактична, вы можете использовать аутентификацию по API-ключу.

**Создание API-ключа:**

1. Откройте десктоп-приложение Rox
2. Перейдите в **Settings > API Keys**
3. Нажмите **Create API Key**
4. Скопируйте ключ (он начинается с `sk_live_`) — увидеть его снова будет невозможно

**Использование API-ключа:**

Передайте API-ключ как Bearer-токен в заголовке `Authorization`:

```
Authorization: Bearer sk_live_your_api_key_here
```

MCP-клиенты, поддерживающие пользовательские заголовки, могут использовать это напрямую. Для Claude Code задайте заголовок в файле `.mcp.json`:

```json
{
  "mcpServers": {
    "rox": {
      "type": "http",
      "url": "https://api.rox.one/api/v2/agent/mcp",
      "headers": {
        "Authorization": "Bearer sk_live_your_api_key_here"
      }
    }
  }
}
```

<Callout type="warn">
API-ключи дают полный доступ к вашей организации. Храните их в секрете и никогда не коммитьте в систему контроля версий. Используйте переменные окружения или менеджеры секретов для CI/CD.
</Callout>

**Когда какой метод использовать:**

| Метод | Подходит для |
|--------|----------|
| OAuth 2.1 | Интерактивное использование — Claude Desktop, Claude Code, Cursor |
| API-ключ | Headless/CI — скрипты, автоматизация, конвейеры |

## Доступные инструменты

### Задачи

| Инструмент | Описание |
|------|-------------|
| `tasks_list` | Список задач с опциональными фильтрами (статус, приоритет, исполнитель, полнотекстовый поиск) |
| `tasks_get` | Получить одну задачу по UUID или slug |
| `tasks_create` | Создать задачу в активной организации |
| `tasks_update` | Обновить поля существующей задачи |
| `tasks_delete` | Жёстко удалить задачу по UUID |

### Рабочие пространства

| Инструмент | Описание |
|------|-------------|
| `workspaces_list` | Список рабочих пространств, опционально с фильтрацией по хосту |
| `workspaces_create` | Создать рабочее пространство на хосте (git worktree, ограниченный веткой) |
| `workspaces_delete` | Удалить рабочее пространство по UUID; идемпотентно |

### Агенты

| Инструмент | Описание |
|------|-------------|
| `agents_list` | Список агентов, настроенных на хосте |
| `agents_create` | Создать (запустить) сессию агента в существующем рабочем пространстве |

### Терминалы

| Инструмент | Описание |
|------|-------------|
| `terminals_create` | Создать сессию терминала в рабочем пространстве, опционально с запуском команды |

### Автоматизации

| Инструмент | Описание |
|------|-------------|
| `automations_list` | Список автоматизаций, принадлежащих вызывающему пользователю |
| `automations_get` | Получить метаданные одной автоматизации (тело промпта опускается) |
| `automations_get_prompt` | Получить полный markdown-промпт одной автоматизации |
| `automations_create` | Запланировать повторяющийся запуск агента с RRULE по RFC 5545 |
| `automations_update` | Обновить метаданные (имя, расписание, агент, хост); используйте `automations_set_prompt` для изменения промпта |
| `automations_set_prompt` | Полностью заменить тело markdown-промпта |
| `automations_pause` | Приостановить автоматизацию (расписание перестаёт срабатывать) |
| `automations_resume` | Возобновить приостановленную автоматизацию |
| `automations_run` | Запустить автоматизацию немедленно, вне расписания |
| `automations_logs` | Список последних запусков автоматизации |
| `automations_delete` | Удалить автоматизацию |

### Проекты

| Инструмент | Описание |
|------|-------------|
| `projects_list` | Список проектов (склонированных репозиториев) в активной организации |

### Хосты

| Инструмент | Описание |
|------|-------------|
| `hosts_list` | Список хостов (зарегистрированных машин), к которым у вас есть доступ |

## Интеграция с чатом

Во встроенной панели чата используйте slash-команду `/mcp`, чтобы увидеть MCP-серверы, настроенные для вашего рабочего пространства, — их имена, тип транспорта (local/remote) и текущее состояние (enabled/disabled/invalid). Это считывается из MCP-конфигурации вашего рабочего пространства (`.mastracode/mcp.json`, `.amp/settings.json` или `.mcp.json`).

## Пример использования

После подключения вы можете попросить вашего AI-агента:

- «Создай задачу для исправления бага со входом»
- «Покажи все назначенные мне задачи»
- «Создай рабочее пространство для функции аутентификации на моём MacBook»
- «Запланируй ежедневную автоматизацию, которая сортирует новые задачи Linear в 9 утра»
- «Приостанови ночную автоматизацию очистки»
- «Покажи последние 10 запусков моей автоматизации сортировки Linear»
