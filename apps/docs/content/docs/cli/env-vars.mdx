---
title: Environment variables
description: Complete reference for environment variables that control Rox CLI behavior.
---

The Rox CLI supports the following environment variables to control
its behavior. Set them in your shell before running `rox`, or
export them from your shell profile (`~/.zshrc`, `~/.bashrc`) or your
CI environment to apply them across every session.

| Variable | Description |
| --- | --- |
| `ROX_API_KEY` | API key (`sk_live_…` / `sk_test_…`) used in place of stored OAuth login. Equivalent to `--api-key`. |
| `ROX_HOME_DIR` | Directory the CLI uses for `config.json` and the host server tree. Default: `~/.rox`. Shared with the desktop app. |
| `GH_TOKEN` / `GITHUB_TOKEN` | Read by the `gh` CLI the host server shells out to for PR checkout and metadata. Set this for sandboxes or CI instead of running `gh auth login` interactively. |

The standalone install script also reads:

| Variable | Description |
| --- | --- |
| `ROX_HOME` | Install root for the standalone CLI and host binary. Default: `~/rox`. |
| `ROX_VERSION` | Release tag to install. Default: `latest`, which resolves through the rolling `cli-latest` release. |
