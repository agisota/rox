Rox CLI (`rox`) — это единый статический бинарник, который работает с тем же
бэкендом, что и десктоп-приложение. Используйте его для управления рабочими пространствами, задачами, автоматизациями
и локальным хост-сервером из терминала или из CI.

## Установка

### Десктоп-приложение

Десктоп-приложение включает Rox CLI. При запуске приложения оно создаёт
управляемый приложением shim по пути `~/.rox/bin/rox`, а терминалы Rox
автоматически добавляют `~/.rox/bin` в начало `PATH`.

Чтобы использовать встроенный CLI из обычной оболочки, добавьте каталог в
профиль вашей оболочки:

```bash
export PATH="$HOME/.rox/bin:$PATH"
```

### Автономная установка

```bash
curl -fsSL https://rox.one/cli/install.sh | sh
```

Скрипт автоматически определяет вашу платформу (macOS Apple Silicon или Linux x86_64),
устанавливает Rox по умолчанию в `~/rox` и добавляет `~/rox/bin`
в `PATH` вашей оболочки.

### Homebrew

```bash
brew install agisota/tap/rox
```

## Вход в систему

```bash
rox auth login
```

CLI запускает локальный callback-сервер на `127.0.0.1:51789` (или `51790`
до `51793`, если предыдущие порты заняты), открывает в браузере страницу
согласия и сохраняет токен сессии в `~/.rox/config.json`.

Если вы состоите в нескольких организациях, вам будет предложено выбрать одну. Чтобы
пропустить запрос и выбрать напрямую:

```bash
rox auth login --organization acme
```

Для CI или других неинтерактивных окружений — где нет браузера, чтобы
завершить OAuth-поток — используйте API-ключ. Вы можете либо сохранить его
один раз с помощью `auth login --api-key`:

```bash
rox auth login --api-key sk_live_…
rox auth whoami
```

…либо задать `ROX_API_KEY` для конкретного вызова, ничего не записывая на диск:

```bash
export ROX_API_KEY=sk_live_…
rox auth whoami
```

API-ключи выдаются на странице **Settings → API Keys** в десктоп-приложении.

## Выполнение первых команд

Выведите список проектов в вашей организации:

```bash
rox projects list
```

Если вашего репозитория там ещё нет, зарегистрируйте его на этой машине — склонируйте из
Git URL или импортируйте существующую копию:

```bash
# Клонировать из URL
rox projects create --name "my-app" --local \
  --clone https://github.com/org/my-app.git \
  --parent-dir ~/code

# Или зарегистрировать существующую локальную копию
rox projects create --name "my-app" --local --import ~/code/my-app
```

Если проект уже существует в вашей организации, но отсутствует на этой машине,
подключитесь к нему вместо создания дубликата:

```bash
rox projects setup <id-from-projects-list> --local --parent-dir ~/code
```

Затем создайте рабочее пространство на его основе:

```bash
rox workspaces create \
  --project prj_… \
  --name "fix-login-bug" \
  --branch fix/login-bug \
  --local
```

Запустите автоматизацию немедленно:

```bash
rox automations list
rox automations run aut_…
rox automations logs aut_…
```

## Режимы вывода

По умолчанию каждая команда выводит представление, удобное для TTY. Два флага меняют
форму вывода для написания скриптов:

| Флаг | Вывод |
| --- | --- |
| `--json` | Полезная нагрузка данных в виде форматированного JSON. Без обёртки. |
| `--quiet` | По одному ID на строку для массивов объектов с `id`; ID для одиночных объектов; иначе JSON. |

```bash
rox tasks list --status in_progress --json | jq '.[].title'
rox workspaces list --quiet | xargs -L1 rox workspaces delete
```

Когда CLI обнаруживает окружение агента или CI (`CLAUDE_CODE`,
`CLAUDECODE`, `CLAUDE_CODE_ENTRYPOINT`, `CODEX_CLI`, `GEMINI_CLI`,
`ROX_AGENT` или `CI`, установленные в непустое значение), по умолчанию используется
вывод JSON, если только вы не передадите `--quiet`.

## Локальные и удалённые хосты

Команды, обращающиеся к хосту, принимают `--host <id>` для выбора хоста, который обработает
запрос, или `--local` для этой машины. Вызовы, направленные локально, идут
напрямую к хост-серверу по loopback (работают офлайн); удалённые
вызовы маршрутизируются через облачный API и реле.

```bash
rox hosts list
rox workspaces list
rox workspaces list --host h_…
rox workspaces list --local
```

Изменяющие команды хоста (`projects create`, `projects setup`,
`workspaces create`) требуют явного `--host` или `--local` — значения
по умолчанию нет. `workspaces list` действует в рамках всей организации, когда ни `--host`,
ни `--local` не указаны; передайте `--local`, чтобы ограничить вывод этой машиной.

## Где хранится состояние

```text
~/.rox/config.json                            # auth, active organization
~/.rox/host/<organizationId>/manifest.json    # host server endpoint + token
~/.rox/host/<organizationId>/host.db          # local host server DB
```

`ROX_HOME_DIR` перемещает всё дерево целиком, совпадая с десктоп-приложением.
