## Обзор

Рабочее пространство существует на машине, где размещены его файлы, терминал и порты. **Удалённые рабочие пространства** позволяют обращаться к рабочему пространству, размещённому на другой машине, — например, на Mac mini под столом, в общей для команды песочнице или на запасном ноутбуке в офисе.

**Rox Relay** — это сервис, обеспечивающий безопасный способ маршрутизации трафика между вашими устройствами. Есть два способа предоставить устройству удалённый доступ через relay:

## Вариант 1 — Desktop

<Callout type="warn">
Открытие вашей основной рабочей станции через relay означает, что всё, доступное вашему локальному хост-сервису — файлы, терминалы, запуски агентов, — становится доступным клиентам, которым вы предоставили доступ. Рекомендуемая настройка — **отдельная машина**, на которой развёрнуты только те репозитории и учётные данные, которыми вы намерены делиться. Так случайный клик или скомпрометированный клиент не сможет стереть посторонний проект или утечь посторонние секреты.

Используйте десктопный переключатель, когда он действительно нужен на этой машине. В остальных случаях предпочитайте путь через CLI на выделенном хосте.
</Callout>

Откройте десктоп-приложение и перейдите в **Settings → Security**. Включите **Allow remote workspaces to access this device via relay**.

![Settings → Security relay toggle](/images/remote-workspaces-security-toggle.png)

Включение переключателя перезапускает хост-сервис, поэтому любые работающие терминалы на этом устройстве будут прерваны. Перед применением изменения вас попросят ввести подтверждающую фразу.

## Вариант 2 — CLI

<Callout type="info">
Чтобы выполнить настройку, см. документацию CLI [Getting Started](/cli/getting-started) и [Host Server](/cli/host-server).
</Callout>

На машине, которую вы хотите добавить в свою общую сеть relay, войдите в систему и запустите хост-сервер.

```bash
rox auth login
rox start --daemon
```

`rox start --daemon` запускает небольшой HTTP-сервер в фоновом режиме и регистрирует машину в relay, чтобы другие клиенты в вашей организации могли её найти.

Проверьте, что она в сети:

```bash
rox status
```

## Предоставление доступа коллегам

Когда хост впервые выходит в сеть, доступ к нему есть только у пользователя, который его запустил. Другие участники вашей организации не увидят его в своём списке рабочих пространств, пока вы их не добавите.

Откройте **Settings → Hosts** в десктоп-приложении, выберите хост и используйте **Add member**, чтобы предоставить доступ коллегам. Владельцы могут повышать участников до владельцев или удалять их с хоста.

![Settings → Hosts members table](/images/remote-workspaces-hosts-members.png)

## Доступ к удалённым рабочим пространствам с другого устройства

На любом другом устройстве, вошедшем в ту же организацию, откройте список **Workspaces** в десктоп-приложении. Используйте фильтр **Device** в правом верхнем углу, чтобы выбрать удалённый хост:

- **This device** — рабочие пространства на машине, которую вы используете сейчас
- **All devices** — всё, к чему у вас есть доступ
- **Other devices** — каждый удалённый хост, который согласился на доступ, с индикатором онлайна

![Device filter in the workspaces list](/images/remote-workspaces-device-filter.png)

Выберите удалённый хост, чтобы увидеть размещённые на нём рабочие пространства. Откройте одно из них — и вы получите полноценный опыт Rox: терминалы, порты, запуски агентов, просмотрщик diff — проксированные через relay к хост-машине.

## Отключение доступа

- **Desktop**: выключите переключатель обратно в **Settings → Security**. Хост-сервис перезапускается и перестаёт принимать трафик relay.
- **CLI**: выполните `rox stop` на хост-машине.

В обоих случаях текущие терминалы и работа на стороне хоста на этой машине прерываются.
