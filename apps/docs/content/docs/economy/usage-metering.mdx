## Один запрос — одна строка

Каждый оплачиваемый запрос к модели записывается отдельной строкой в `usage_requests`: какая модель, сколько токенов на входе и выходе, итоговая стоимость в USD и в Rox, плюс трассировка (`trace`) для аудита.

## Каталог моделей

Цены берутся из каталога моделей, синхронизируемого с публичными прайсами (источник — models.dev). Каждая модель хранит публичную цену за миллион входных и выходных токенов и относится к ценовому семейству.

## Как считается стоимость

Цена продажи Rox выводится из публичной цены модели, а не задаётся вручную:

1. Берётся **публичная цена за миллион токенов** (USD) из каталога.
2. Она делится на **делитель ценового семейства** — это рычаг маржи, не вычисляемая ставка.
3. Результат (USD за миллион) переводится в Rox по курсу **1 USDT = 100 Rox**.
4. Итоговая стоимость пропорциональна числу токенов запроса.

Текущие делители по семействам:

| Семейство | Примеры моделей | Делитель |
| --- | --- | --- |
| `xai` | Grok | ÷ 7.5 |
| `openai` | GPT, o-серия | ÷ 7.5 |
| `anthropic` | Claude (Opus / Sonnet) | ÷ 5.25 |
| `google` | Gemini | ÷ 12.25 |
| `other` | deepseek, cohere, kimi, mistral, minimax, … | ÷ 25 |

<Callout type="info">
Делители — это настраиваемый параметр маржи (issue #34), а не привязанная к измеренной себестоимости величина. Они пересматриваются по заданной периодичности и могут меняться; всегда сверяйтесь с актуальным каталогом, а не с этой таблицей.
</Callout>

Все суммы Rox квантуются до 6 знаков после запятой, чтобы расчёт в коде совпадал с тем, что хранит Postgres, и журнал сходился бит в бит.

## API чтения потребления

Сводку по потреблению (итоги, разбивка по инструментам, дневная динамика за 30 дней) отдаёт публичная процедура `usage.summary` (профиль должен быть публичным или это ваш собственный профиль). Клиенты (CLI, агенты) отправляют пакеты строк потребления через защищённую процедуру `usage.recordBatch`.
