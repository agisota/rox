## Обзор

Запускайте AI-агентов для программирования в изолированных рабочих пространствах. Каждый агент работает независимо, не мешая другим.

![AI Agents](/images/agents.png)

## Поддерживаемые агенты

- **Amp** — Amp Code CLI
- **Claude Code** — CLI от Anthropic
- **Codex** — ассистент от OpenAI
- **OpenCode** — альтернатива с открытым исходным кодом
- **Pi** — минималистичный терминальный инструмент для программирования
- **Gemini CLI** — CLI-агент от Google
- **Cursor Agent** — AI-агент Cursor

## Чат

Rox включает встроенную панель чата для взаимодействия с AI, не покидая приложение.

- Используйте выбор модели, чтобы выбрать поддерживаемые модели и подключить провайдеров
- Заголовки сессий чата генерируются автоматически из разговора
- Используйте управление уровнем размышления, чтобы задать усилие рассуждения: **Off**, **Low**, **Medium**, **High** или **Max**

## Настройки агентов

Перейдите в **Settings → Agents**, чтобы настроить поведение запуска для каждого встроенного агента.

- **Enabled** — показать или скрыть агента в средствах запуска
- **Command (No Prompt)** — базовая команда для терминальных агентов
- **Command (With Prompt)** — команда с учётом промпта для терминальных агентов
- **Prompt Command Suffix** — необязательные флаги, добавляемые после полезной нагрузки промпта
- **Task Prompt Template** — шаблон, используемый при запуске агента из задачи
- **Model Override** — необязательное переопределение модели для Rox Chat
- **Reset to Defaults** — восстановить поставляемую конфигурацию для этого агента

## Использование

Запускайте агентов прямо в терминале:

```bash
claude "Add error handling to fetchUser"
```

## Команда автозапуска

При запуске агента из задачи переключатель **Auto-run command** определяет, выполняется ли она немедленно.

- **On** — терминальные агенты сразу выполняют свою команду, а Rox Chat немедленно отправляет промпт задачи
- **Off** — терминальные агенты подготавливают команду без нажатия Enter, а Rox Chat открывается как черновик

Эта настройка запоминается между сессиями.

## Параллельные агенты

Создавайте несколько рабочих пространств и запускайте агента в каждом. Сравнивайте результаты и объединяйте лучшее решение.

## Встроенные промпты

Агенты могут приостанавливаться и задавать вопросы, запрашивать подтверждение инструментов и просить о ревью плана прямо в панели чата.

## Доступ к песочнице

Когда агенту нужен доступ к файлам за пределами его рабочего пространства, он запросит разрешение в чате, чтобы вы могли подтвердить или отклонить запрос на месте.

## Уведомления

Получайте уведомления, когда агенты завершают работу. Настройте звуки в Settings.
