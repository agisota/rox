## Rox как OAuth-провайдер

Помимо входа пользователей, Rox умеет выступать **OAuth/OIDC-провайдером** для внешних клиентов (например, для MCP-агентов, которым нужен доступ к API от имени пользователя). Это обеспечивает плагин `oauthProvider` поверх better-auth.

Ключевые свойства:

- поддерживается **динамическая регистрация клиентов** (в т.ч. без предварительной аутентификации клиента);
- страница входа — `/sign-in`, страница согласия — `/oauth/consent` в веб-приложении;
- access-токены живут **7 дней**;
- валидные audience включают сам API и MCP-транспорты (`/api/agent/mcp`, `/api/v2/agent/mcp`).

## Эндпоинты обнаружения (.well-known)

Метаданные провайдера публикуются по стандартным `.well-known`-путям на API-хосте (`https://api.rox.one`):

| Эндпоинт | Назначение |
| --- | --- |
| `/.well-known/openid-configuration` | OpenID Connect discovery — список эндпоинтов и возможностей. |
| `/.well-known/oauth-authorization-server` | Метаданные OAuth 2.0 authorization server. |
| `/.well-known/oauth-protected-resource` | Метаданные защищённого ресурса (resource server). |

Открытый ключ для проверки подписи токенов доступен через JWKS-эндпоинт better-auth (`/api/auth/jwks`).

## Поток согласия

После входа пользователь попадает на страницу согласия `/oauth/consent`. Там же выбирается организация: выбранная организация становится `referenceId` токена, а полный список членств пользователя добавляется в claim `organizationIds` — без этого проверки доступа на стороне API отклоняли бы токен.

## Claims токенов

Как session-JWT, так и OAuth access-токены несут согласованный набор claims:

| Claim | Значение |
| --- | --- |
| `sub` | Идентификатор пользователя. |
| `email` | Email пользователя. |
| `organizationIds` | Все организации, в которых состоит пользователь. |
| `organizationId` | Активная/выбранная организация (для OAuth). |

<Callout type="info">
Session-JWT (плагин `jwt`) и OAuth-токены подписываются одной парой ключей RS256, поэтому любой сервис, который проверяет токены через публичный JWKS (включая relay), принимает их одинаково.
</Callout>
