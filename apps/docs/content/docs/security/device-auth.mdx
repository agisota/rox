## Зачем отдельный поток для устройств

Десктоп-приложение и CLI — это не браузер: им нужно получить и хранить долгоживущие учётные данные, чтобы обращаться к API без cookie-сессии. Rox использует для этого подключение устройства (device connect) и Bearer-токены.

## Подключение десктопа

Десктоп-приложение проходит вход через веб-поток и затем привязывает учётные данные к приложению:

- веб-поверхность входа десктопа — `/auth/desktop`;
- эндпоинт привязки на API — `/api/auth/desktop/connect`;
- десктоп обращается к нашему API через клиента с Bearer-токеном (`apiTrpcClient`), используя те же учётные данные.

Десктоп доверяется как источник через схемы `rox://app` / `rox://` в `trustedOrigins`.

## Вход CLI

CLI авторизуется через веб-поток `/cli/auth` либо использует API-ключ напрямую:

```bash
rox auth login          # интерактивный вход через браузер
# или
export ROX_API_KEY=sk_live_xxx
```

Подробнее см. [CLI Getting Started](/cli/getting-started) и [Переменные окружения](/cli/env-vars).

## Токены хостов (bootstrap)

Когда провизионится десктоп/хост, который должен «дозвониться» до relay, ему выдаётся **короткоживущий подписанный JWT**, выпущенный API (повторно используется JWKS/jwt better-auth). Токен ограничен полями `{ hostId, userId, exp }` с коротким TTL (минуты — ровно чтобы загрузиться и подключиться). Relay проверяет его при регистрации туннеля тем же способом, что и любой другой Rox-JWT, — отдельного верификатора нет.

<Callout type="info">
Все токены Rox — session-JWT, OAuth-токены и bootstrap-токены хостов — подписаны одной парой ключей RS256. Любой сервис, проверяющий через публичный JWKS (`/api/auth/jwks`), включая relay, принимает их единообразно.
</Callout>

## Безопасность relay

[Удалённые рабочие пространства](/remote-workspaces) маршрутизируют трафик между вашими устройствами через **Rox Relay**. Важная оговорка по безопасности:

<Callout type="warn">
Открытие машины через relay делает всё, к чему имеет доступ её локальный хост-сервис — файлы, терминалы, запуски агентов, — доступным для клиентов, которым вы выдали доступ. Рекомендуется **отдельная машина**, на которой развёрнуты только те репозитории и учётные данные, которыми вы готовы делиться. Так случайный клик или скомпрометированный клиент не сможет стереть посторонний проект или утечь посторонние секреты.
</Callout>

Когда хост впервые выходит в сеть, доступ к нему есть только у запустившего его пользователя. Остальные участники организации не увидят хост, пока вы не добавите их через **Settings → Hosts → Add member**. Отключение — выключить переключатель в **Settings → Security** (Desktop) или выполнить `rox stop` (CLI).
