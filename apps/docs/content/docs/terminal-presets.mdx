## Обзор

Пресеты позволяют быстро запускать терминалы с заранее настроенными командами. Сохраняйте типовые рабочие процессы и применяйте их в один клик.

## Что делают пресеты

- **Пользовательские команды для новых вкладок** — задайте команды, которые выполняются при открытии терминала
- **Рабочий каталог** — укажите, где выполняются команды в вашем проекте (например, `apps/web`)
- **Режим запуска: разделение панели или новая вкладка** — выполняйте команды параллельно в одной вкладке или по одной вкладке на команду
- **Пресет по умолчанию** — автоматическое применение пресета к новым рабочим пространствам
- **Шаблоны быстрого добавления** — настройка в один клик для Claude, Codex, Gemini и других агентов

## Создание пресета

Перейдите в **Settings → Terminal** и нажмите **Add Preset**.

Каждый пресет содержит:

| Поле | Описание |
|-------|-------------|
| **Name** | Отображаемое имя пресета |
| **Description** | Необязательное описание |
| **Working Directory** | Относительный путь в вашем проекте (оставьте пустым для корня) |
| **Commands** | Одна или несколько команд для выполнения |
| **Mode** | Split Pane (параллельные панели) или New Tab (параллельные вкладки) |

## Режимы запуска

**Split Pane** — команды выполняются параллельно в одной вкладке, по одной разделённой панели на команду.

**New Tab** — команды выполняются параллельно в отдельных вкладках, по одной вкладке на команду.

Пресеты по умолчанию выполняются параллельно.

## Шаблоны быстрого добавления

Заранее настроенные пресеты для популярных ИИ-агентов. Значения по умолчанию повторяют встроенные настройки запуска Rox для каждого агента. Claude и Codex используют свои наиболее разрешительные флаги обхода; отредактируйте любой пресет, чтобы изменить режим разрешений.

- **amp** — `amp` (встроенные правила разрешений автоматически отклоняют деструктивные операции)
- **claude** — `claude --dangerously-skip-permissions`
- **codex** — `codex --dangerously-bypass-approvals-and-sandbox` (наиболее разрешительный)
- **gemini** — `gemini --approval-mode=auto_edit`
- **copilot** — `copilot --allow-tool=write`
- **cursor-agent** — `cursor-agent` (запрашивает подтверждение для каждого действия)
- **mastracode** — агент-кодер от Mastra (opt-in: по умолчанию автоматически одобряет все действия на уровне CLI; нет флага запуска для ограничения)
- **opencode** — ИИ-агент-кодер с открытым исходным кодом (opt-in: по умолчанию полный доступ к файлам и shell на уровне CLI; нет флага запуска для ограничения)
- **pi** — минимальная терминальная среда для кодинга (opt-in: по умолчанию автоматически одобряет все действия на уровне CLI; нет флага запуска для ограничения)

## Панель пресетов

Панель пресетов появляется над вкладками терминала и показывает закреплённые пресеты для доступа в один клик.

- **Закрепить пресет** — используйте элемент управления закреплением в **Settings → Terminal** или меню панели пресетов
- **Запуск** — нажмите на закреплённый пресет, чтобы открыть его в текущем рабочем пространстве
- **Правый клик** — откройте в текущем терминале, текущей вкладке или новой вкладке либо отредактируйте пресет
- **Изменение порядка** — перетаскивайте закреплённые элементы, чтобы переупорядочить их на панели
- **Управление** — используйте меню с шестерёнкой, чтобы просматривать пресеты, добавлять шаблоны быстрого добавления и показывать или скрывать панель

## Пресет по умолчанию

Отметьте пресет как используемый по умолчанию, чтобы он автоматически применялся при создании рабочих пространств или открытии новых вкладок.

**Настройки автоматического применения** в Settings → Terminal:
- **Apply preset on workspace creation** — выполняет пресет по умолчанию при создании новых рабочих пространств
- **Apply preset on new tab** — выполняет пресет по умолчанию при открытии новых вкладок/панелей/разделений терминала

Обе настройки включены по умолчанию. Отключите любую из них, чтобы предотвратить автоматическое применение пресета в этом контексте.

Явные запуски пресетов (через горячую клавишу или меню) всегда работают независимо от этих настроек.
