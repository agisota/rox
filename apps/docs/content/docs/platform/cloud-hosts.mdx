## Обзор

**Облачный хост** — это машина, на которой работает только хост-сервис Rox, без UI: выделенный сервер, Mac mini под столом, командная песочница или CI-раннер. Клиенты ([веб](/platform/web-app), [мобайл](/platform/mobile), [десктоп](/platform/hybrid-host)) подключаются к нему через [Rox Relay](/self-host/overview).

<Callout type="info">
Облачный хост — это та же роль «хост», что встроена в десктоп, но запущенная отдельно через CLI. Подходит, когда вы хотите запускать агентов и рабочие пространства на машине, отличной от той, где открыт клиент.
</Callout>

## Как поднять

На headless-машине запускается автономный хост-сервер командой `rox start` (см. [Хост-сервер](/cli/host-server)). Он управляет рабочими пространствами, портами и запусками агентов на этой машине.

| Требование | Назначение | Источник |
| --- | --- | --- |
| `git` в `PATH` | clone, worktree, ветки | `apps/docs/content/docs/cli/host-server.mdx` |
| `gh` в `PATH` (или `GH_TOKEN`) | checkout PR, метаданные | `cli/host-server` |
| Аутентификация Rox CLI | хост работает под вашей сессией | `rox auth login` |
| Агентские CLI (опционально) | запуск агентов в рабочих пространствах | по наличию в `PATH` |

## Доступ через relay

Rox Relay (`apps/relay`) безопасно маршрутизирует трафик между клиентами и хостами. Клиент адресует хост по его routing key:

```
Клиент → relay.rox.one/hosts/<routingKey>/trpc/<procedure> → host-service
```

Relay проверяет доступ (`apps/relay/src/access.ts`) и JWT (`apps/relay/src/auth.ts`), затем туннелирует запрос к нужному хосту (`apps/relay/src/tunnel.ts`). Хост должен быть в сети.

<Callout type="warn">
Хост делает доступными свои файлы, терминалы и запуски агентов всем клиентам, которым предоставлен доступ. Размещайте на облачном хосте только те репозитории и учётные данные, которыми намерены делиться.
</Callout>
