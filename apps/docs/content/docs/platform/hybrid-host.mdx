## Обзор

Десктоп-приложение (`@rox/desktop`, `apps/desktop`) — это **гибрид**: оно одновременно клиент (рендерит UI) и хост (запускает встроенный хост-сервис на вашей машине). Поэтому десктоп работает локально и офлайн: рабочие пространства, терминалы и запуски агентов выполняются прямо на устройстве, без relay.

<Callout type="info">
Десктоп построен на Electron. Он включает CLI и устанавливает shim `~/.rox/bin/rox` при запуске, поэтому команды `rox` доступны в терминалах Rox автоматически.
</Callout>

## Две роли в одном приложении

| Роль | Что делает | Источник |
| --- | --- | --- |
| **Клиент** | Рендерит UI, открывает рабочие пространства, diff, чат | `apps/desktop/src/renderer` |
| **Хост** | Запускает встроенный host-service: worktree, PTY, агенты | `LocalHostServiceProvider` + `@rox/host-service` |

Рендерер подключается к встроенному хост-сервису через **локальный транспорт** общего контракта `HostClient` (`packages/shared/src/host-client/`) — тот же контракт, что веб использует поверх relay. Благодаря этому код экранов одинаков для локального и удалённого хостов.

```
Десктоп-приложение (Electron)
   ├── Renderer (клиент)
   │      └── HostClient (LocalTransport)
   │                  │ (внутри процесса)
   └── host-service (хост) → worktree · терминалы · агенты
```

## Совместное использование с relay

Десктоп может опционально открыть свой встроенный хост для удалённого доступа через relay (**Settings → Security → Allow remote workspaces to access this device via relay**). Тогда другие клиенты (веб, мобайл, другой десктоп) могут работать с рабочими пространствами на этой машине.

<Callout type="warn">
Открытие основной рабочей станции через relay делает доступными клиентам всё, что доступно вашему хост-сервису — файлы, терминалы, запуски агентов. Для общего доступа предпочитайте выделенную машину или [облачный хост](/platform/cloud-hosts) на CLI. См. [Удалённые рабочие пространства](/remote-workspaces).
</Callout>
