## Обзор

Rox — это единое ядро (`@rox/trpc`, `@rox/host-service`, `@rox/shared`) с несколькими клиентскими поверхностями поверх него. Рабочее пространство всегда живёт на **хосте** — машине, где находятся его файлы, терминалы и порты. Клиенты подключаются к хостам напрямую (локально) или через [Rox Relay](/self-host/overview).

<Callout type="info">
Ключевое разделение — **клиент** и **хост**. Десктоп совмещает обе роли; веб и мобайл — это только клиенты; облачный хост — это только хост (headless).
</Callout>

## Поверхности

| Платформа | Пакет | Роль | Источник |
| --- | --- | --- | --- |
| **Веб-приложение** | `@rox/web` | Только клиент (app.rox.one) | `apps/web` |
| **Десктоп** | `@rox/desktop` | Клиент + хост (гибрид) | `apps/desktop` |
| **Мобайл** | `@rox/mobile` | Только клиент (Expo) | `apps/mobile` |
| **Облачный хост** | `@rox/host-service` + CLI | Только хост (headless) | `packages/host-service`, `apps/relay` |

## Контракт хост-клиента

Все клиенты обращаются к хостам через общий контракт `HostClient` (`packages/shared/src/host-client/`). Это даёт единое поведение независимо от транспорта:

- **Локальный транспорт** — десктоп говорит со своим встроенным хост-сервисом напрямую (внутри процесса).
- **Relay-транспорт** — веб говорит с удалёнными хостами через relay (`apps/web/src/trpc/host-client.ts`).

```
Клиент (web / desktop / mobile)
        │
        ├── HostClient (общий контракт, packages/shared/src/host-client)
        │       ├── LocalTransport  → встроенный host-service (десктоп)
        │       └── RelayTransport  → relay → удалённый host-service
        │
        ▼
Хост (host-service): рабочие пространства, терминалы, запуски агентов
```
