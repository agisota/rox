## Обзор

Веб-приложение (`@rox/web`, `apps/web`) — это основная клиентская поверхность Rox по адресу **app.rox.one**. Это **только клиент**: у него нет локального хост-сервиса. Все операции, привязанные к машине разработчика — файлы, терминалы, запуски агентов, — маршрутизируются к [хосту](/platform/cloud-hosts) через [Rox Relay](/self-host/overview).

<Callout type="info">
Веб-приложение построено на Next.js 16. Перехват запросов реализован через `apps/web/src/proxy.ts` (в Next.js 16 `middleware.ts` переименован в `proxy.ts`).
</Callout>

## Транспорт к хостам

Веб реализует общий контракт `HostClient` (`packages/shared/src/host-client/`) через **relay-транспорт** (`apps/web/src/trpc/host-client.ts`). Облачные данные (задачи, проекты, организация) идут напрямую к `api.rox.one`; хостовые операции проходят через relay к нужной машине.

| Тип операции | Маршрут | Пример |
| --- | --- | --- |
| Облачные данные | `api.rox.one` (tRPC) | задачи, проекты, организация |
| Операции на хосте | relay → host-service | рабочие пространства, терминалы, чат, diff |

```
Браузер (app.rox.one)
   ├── tRPC → api.rox.one        (облачные данные)
   └── HostClient (RelayTransport) → relay → host-service   (операции на хосте)
```

## Кабинет агентов

Веб привязывает кабинет агентов к реальным хостам через relay: онлайн-хосты перечисляются, рабочие пространства и сессии открываются на выбранной машине. Если у организации нет онлайн-хостов, поверхность показывает offline-состояние вместо ложного пустого списка.

## Аутентификация

Веб использует cookie-сессии better-auth (см. [Модель аутентификации](/security/auth-model)). Доступ к хостам через relay авторизуется общим JWT (см. [Удалённые рабочие пространства](/remote-workspaces)).
