## Обзор

Мобильное приложение (`@rox/mobile`, `apps/mobile`) — это клиент Rox на **Expo / React Native**. Как и [веб](/platform/web-app), это только клиент: операции на хостах маршрутизируются через relay, а данные организации синхронизируются в реальном времени.

<Callout type="info">
Маршрутизация — на expo-router. Структура повторяет паттерн репозитория: маршруты в `app/` переэкспортируют компоненты из `screens/`, зеркаля структуру каталогов (см. `apps/mobile/AGENTS.md`).
</Callout>

## Cache-first синхронизация данных

Мобайл читает данные организации через **Electric-коллекции** (`apps/mobile/lib/collections/collections.ts`) поверх TanStack DB. Коллекции подписываются на шейпы Electric:

```
EXPO_PUBLIC_API_URL/api/electric/v1/shape
```

Коллекции **cache-first**: `useLiveQuery` может вернуть сохранённые строки в `data`, пока коллекция ещё не `isReady`. Существующие строки рендерятся сразу; `isReady` влияет только на то, что показать при отсутствии данных:

| Состояние | Что показать |
| --- | --- |
| есть `data` | рендерить строки (даже если `isReady` ещё `false`) |
| нет `data` + не `isReady` | загрузка / скелетон / `null` |
| нет `data` + `isReady` | пусто / не найдено |

<Callout type="warn">
Никогда не скрывайте и не заменяйте существующие `data` только потому, что `isReady` или `isLoading` ещё в процессе. Строгую готовность ждут только перед записью/сидингом значений по умолчанию.
</Callout>

## Поверхности

| Группа маршрутов | Назначение | Источник |
| --- | --- | --- |
| `app/(auth)` | Вход и регистрация | `apps/mobile/app/(auth)` |
| `app/(authenticated)` | Задачи, проекты, рабочие пространства | `apps/mobile/app/(authenticated)` |

Экраны переиспользуют те же доменные модели и контракты, что веб и десктоп, через общие пакеты (`@rox/trpc`, `@rox/db`, `@rox/shared`) — единый core, платформенные адаптеры на краю.
