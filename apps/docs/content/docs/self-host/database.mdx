## Postgres: Neon или свой

Rox использует Postgres через Drizzle ORM. Есть два варианта развёртывания:

| Вариант | Когда использовать | Как |
| --- | --- | --- |
| **Neon (managed)** | Продакшен, отсутствие хлопот с эксплуатацией | Заполните `DATABASE_URL` / `DATABASE_URL_UNPOOLED` строками Neon. |
| **Self-host Postgres** | Локальная разработка, полный контроль | `docker-compose.yml` поднимает `postgres:17` + `neon-proxy`. |

Локальный **neon-proxy** (`ghcr.io/timowilhelm/local-neon-http-proxy`) эмулирует HTTP-интерфейс Neon, чтобы драйвер Drizzle работал одинаково и локально, и в облаке.

<Callout type="info">
Postgres должен работать с `wal_level=logical` (как в `docker-compose.yml`), иначе ElectricSQL не сможет подписываться на изменения.
</Callout>

## ElectricSQL — живая синхронизация

ElectricSQL (`electricsql/electric:1.4.13`) обеспечивает живую синхронизацию данных уровня организации/аккаунта (задачи, участники, организации) в клиенты через shape-подписки. Для HTTPS-потоков Electric используется Caddy как reverse-proxy (`Caddyfile.example`).

Хост-ориентированные живые данные (терминалы, git, файловая система, чат) читаются **не через Electric**, а через прикреплённый хост по транспорту relay — хост является единственным источником истины для своих данных. Electric отвечает только за durable-данные уровня организации.

## Миграции

<Callout type="warn">
Никогда не редактируйте файлы в `packages/db/drizzle/` вручную — `.sql`-миграции, `meta/_journal.json` и снапшоты генерируются автоматически. Меняйте только схему в `packages/db/src/schema/`.
</Callout>

Рабочий процесс:

1. Измените схему в `packages/db/src/schema/`.
2. Сгенерируйте SQL **офлайн** (diff схемы против снапшота — БД не затрагивается):

   ```bash
   bunx drizzle-kit generate --name="<имя_в_snake_case>"
   ```

3. Применение миграций (`drizzle-kit migrate` / `push`) — это **шаг деплоя**. Агенты его не выполняют; против продакшена он запускается только в явном, человеко-гейтированном процессе деплоя.

Для локального теста применённой миграции поднимайте отдельную ветку Neon и направляйте корневой `.env` на неё — никогда на продакшен.
