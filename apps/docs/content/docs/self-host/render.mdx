## Обзор

Продакшен-сервисы Rox разворачиваются на **Render**. Каждое приложение монорепозитория — отдельный сервис Render с автодеплоем из ветки `main`.

<Callout type="info">
Это справочное описание текущей продакшен-конфигурации владельца. Для собственного self-host можно использовать любую платформу — Render не обязателен. Альтернатива без платформы — [Docker Compose](/self-host/docker-compose).
</Callout>

## Ручной деплой

Когда нужно задеплоить конкретный сервис на известный коммит, используется вспомогательный скрипт:

```bash
scripts/render-deploy.sh marketing
scripts/render-deploy.sh marketing <commit-sha>
```

Под капотом он выполняет:

```bash
render deploys create <service-id> [--commit <sha>] --wait --confirm
```

## Проверка

Используйте Render CLI как источник истины по сервисам:

```bash
render services --output json
render deploys list <service-id> --output json
```

Публичный smoke-тест продакшена — проверка реального публичного хоста:

```bash
curl -I https://rox.one
```

## Миграции базы данных

<Callout type="warn">
Старый продакшен-workflow запускал миграции БД перед деплоем — он отключён. Запускайте миграции только через явный процесс миграции целевого окружения; не возвращайте неявные продакшен-миграции в деплой-workflow.
</Callout>
