## Client

```ts
import Rox from '@rox_sh/sdk';

const client = new Rox({
  apiKey: 'sk_live_…',
  organizationId: '…',
  // optional:
  baseURL: 'https://api.rox.one',
  relayURL: 'https://relay.rox.one',
  timeout: 60_000,
  maxRetries: 2,
  logLevel: 'warn', // 'off' | 'error' | 'warn' | 'info' | 'debug'
});
```

## tasks

### `tasks.create(body)`

Создаёт задачу.

```ts
const task = await client.tasks.create({
  title: 'Wire up auth',
  description: 'See SUPER-100',
  priority: 'high',          // 'urgent' | 'high' | 'medium' | 'low' | 'none'
  assigneeId: '<uuid>',      // optional
  statusId: '<uuid>',        // optional, defaults to first backlog status
  estimate: 4,               // optional, story points
  dueDate: '2026-12-31',     // optional, ISO date
  labels: ['bug'],           // optional
});
```

Возвращает `Task`.

### `tasks.list(params?)`

Возвращает список задач с расширенными фильтрами. Все фильтры комбинируются по логике AND.

```ts
const tasks = await client.tasks.list({
  assigneeMe: true,          // tasks assigned to you
  creatorMe: true,           // tasks you created
  priority: 'high',
  statusId: '<uuid>',
  assigneeId: '<uuid>',      // someone else's tasks
  search: 'auth',            // substring of title
  limit: 50,                 // max 500
  offset: 0,
});
```

Возвращает `Array<TaskListItem>` — каждая строка представляет собой `Task`, денормализованный с полями `assigneeName`, `assigneeImage`, `creatorName`, `creatorImage` и `statusName`, так что вам не нужны дополнительные вызовы для отображения.

### `tasks.retrieve(idOrSlug)`

Находит одну задачу по id или slug.

```ts
const task = await client.tasks.retrieve('SUPER-172');
if (!task) throw new Error('not found'); // returns null when missing
```

Возвращает `Task | null`.

### `tasks.update(body)`

Частично обновляет задачу.

```ts
const task = await client.tasks.update({
  id,
  priority: 'urgent',
  statusId: '<uuid>',
  prUrl: 'https://github.com/…',
});
```

Возвращает обновлённый `Task`.

### `tasks.delete(id)`

Мягко удаляет задачу.

```ts
await client.tasks.delete(id);
```

### `tasks.statuses.list()`

Возвращает список статусов задач (состояний рабочего процесса), настроенных для активной организации, в порядке отображения. Полезно для определения id статусов перед [`tasks.create`](#taskscreatebody) или [`tasks.update`](#tasksupdatebody).

```ts
const statuses = await client.tasks.statuses.list();
const todo = statuses.find((s) => s.type === 'unstarted');
```

Возвращает `Array<TaskStatus>` — каждый элемент имеет вид `{ id, name, color, type, position }`.

## workspaces

### `workspaces.list(params?)`

Возвращает список рабочих пространств в организации.

```ts
const all = await client.workspaces.list();
const onOneHost = await client.workspaces.list({ hostId: '<machineId>' });
```

Возвращает `Array<Workspace>`.

### `workspaces.create({ hostId, projectId, name, branch, agents?, command? })`

Создаёт worktree на конкретном хосте. Опционально запускает одного или нескольких агентов внутри него сразу, как только worktree готов, и/или выполняет разовую shell-команду `command` в worktree. Проходит через relay-туннель — хост должен быть в сети.

Каждый элемент в `agents` принимает id пресета (например, `"claude"`) или UUID экземпляра `HostAgentConfig` — служба хоста сопоставляет его с настроенными строками и копирует их сохранённые `command`, `args` и `env`. Используйте [`agents.list({ hostId })`](#agentslist-hostid-), чтобы перечислить установленные. `command` не зависит от `agents` — передавайте любое из них или оба.

```ts
const result = await client.workspaces.create({
  hostId: '<machineId>',     // see hosts.list()
  projectId: '<uuid>',       // see projects.list()
  name: 'wire-up-auth',
  branch: 'feat/auth',
  agents: [                  // optional — spawn agents on creation
    { agent: 'claude', prompt: 'Implement the auth flow described in SUPER-100.' },
    { agent: 'claude', prompt: 'Write integration tests for the new auth flow.' },
  ],
});

console.log(result.workspace.id);
for (const agent of result.agents) {
  if (agent.ok) console.log(agent.kind, agent.sessionId, agent.label);
  else console.error(agent.error);
}
```

Возвращает `WorkspaceCreateResult`:

- `workspace` — созданная строка рабочего пространства (`id`, `name`, `branch`, `hostId`, `projectId`, `type`, `taskId`, …).
- `terminals` — терминалы, открытые на хосте (`{ terminalId, label? }`).
- `agents` — по одному элементу на каждого агента в запросе (пустой массив, если `agents` был опущен). Каждый элемент имеет вид либо `{ ok: true, kind: 'terminal' | 'chat', sessionId, label }`, либо `{ ok: false, error }`.

### `workspaces.update(id, params)`

Обновляет поля существующего рабочего пространства. Перенос рабочего пространства на другую ветку или хост требует оркестрации на стороне хоста и небезопасен для управления напрямую через облачный API.

```ts
await client.workspaces.update('<workspaceId>', { name: 'wire-up-auth-v2' });
```

Возвращает обновлённую строку рабочего пространства (`id`, `name`, `branch`, `hostId`, `projectId`, `type`, `taskId`, …).

### `workspaces.delete(id, opts?)`

Удаляет рабочее пространство. Автоматически находит его хост и маршрутизирует через relay.

```ts
await client.workspaces.delete(id);
// or skip the lookup:
await client.workspaces.delete(id, { hostId: '<machineId>' });
```

## hosts

### `hosts.list()`

Возвращает список машин разработчиков, зарегистрированных в организации. Возвращает `Array<Host>` с полями `id` (machineId), `name`, `online` и `organizationId`.

```ts
const hosts = await client.hosts.list();
const online = hosts.filter((h) => h.online);
```

## agents

Строки терминальных агентов, настроенные на конкретном хосте (те же строки, что показаны в
Settings → Agents на этой машине). Чтение проходит через relay-туннель —
целевой хост должен быть в сети.

### `agents.list({ hostId })`

Возвращает список настроенных агентов на хосте в сохранённом порядке отображения. Первый вызов на
свежем хосте инициализирует встроенные значения по умолчанию. Возвращает `Array<HostAgentConfig>` —
каждая строка содержит `id`, `presetId`, `label`, `command`, `args`, `env` и
поля транспорта промптов, используемые для запуска агента.

```ts
const agents = await client.agents.list({ hostId: '<machineId>' });
```

### `agents.create({ workspaceId, agent, prompt, attachmentIds? }, { hostId? })`

Создаёт (запускает) сессию агента в существующем рабочем пространстве. По умолчанию находит
хост, которому принадлежит рабочее пространство, через облачный индекс; передайте явный `hostId`,
чтобы пропустить поиск.

`agent` принимает либо id пресета (например, `"claude"`), либо UUID экземпляра
`HostAgentConfig`. Возвращает `{ kind, sessionId, label }`.

```ts
const { sessionId } = await client.agents.create({
  workspaceId: '<uuid>',
  agent: 'claude',
  prompt: 'Audit the login flow for race conditions.',
});
```

## terminals

Терминальные (PTY) сессии на хосте, привязанные к рабочему пространству.

### `terminals.create({ workspaceId, command?, cwd? }, { hostId? })`

Создаёт терминальную сессию в существующем рабочем пространстве. По умолчанию находит
хост, которому принадлежит рабочее пространство, через облачный индекс; передайте явный `hostId`,
чтобы пропустить поиск. Передайте `command`, чтобы выполнить разовую shell-команду, или опустите его,
чтобы открыть интерактивную оболочку. Возвращает `{ terminalId, status }`.

```ts
const { terminalId } = await client.terminals.create({
  workspaceId: '<uuid>',
  command: 'bun install && bun test',
});
```

## automations

Регулярные запуски агентов по расписанию RRULE. Требует подписку Pro для организации для `create` / `update` / `delete`.

### `automations.list(params?)`

```ts
const automations = await client.automations.list();
// or filter by case-insensitive substring on name:
const triage = await client.automations.list({ name: 'triage' });
```

Каждая строка представляет собой `AutomationSummary` — тело `prompt` опущено (оно может быть
большим markdown). Получите его с помощью `automations.getPrompt(id)`. Каждая строка содержит
`scheduleText` — человекочитаемое представление rrule.

### `automations.retrieve(id)`

```ts
const a = await client.automations.retrieve(id);
```

Возвращает `AutomationSummary` (без тела `prompt` — вызовите `getPrompt(id)`).

### `automations.create(body)`

Создаёт регулярную автоматизацию.

```ts
const a = await client.automations.create({
  name: 'Daily leads',
  prompt: 'Find new leads from Linear and update the CRM…',
  agent: 'claude', // host agent presetId, instance UUID, or 'rox' for built-in chat
  rrule: 'FREQ=DAILY;BYHOUR=6;BYMINUTE=0',
  timezone: 'America/Los_Angeles',
  v2ProjectId: '<uuid>',     // one of v2ProjectId or v2WorkspaceId required
  // optional:
  v2WorkspaceId: '<uuid>',   // pin to a specific workspace
  targetHostId: '<machineId>', // pin to a specific host
  dtstart: new Date().toISOString(),
  mcpScope: ['linear', 'notion'],
});
```

### `automations.update(body)`

```ts
await client.automations.update({ id, rrule: 'FREQ=WEEKLY;BYDAY=MO' });
```

### `automations.delete(id)`

### `automations.run(id)`

Запускает немедленно, вне расписания. Проходит через relay к закреплённому хосту.

```ts
const run = await client.automations.run(id);
console.log(run.status, run.hostId);
```

### `automations.pause(id)` / `automations.resume(id)`

Переключает флаг `enabled`. Приостановленная автоматизация перестаёт запускаться по своему расписанию, пока вы её не возобновите.

### `automations.logs(id, params?)`

История запусков. Только для владельца — возвращает 404, если автоматизация не принадлежит вызывающему пользователю.

```ts
const runs = await client.automations.logs(id, { limit: 20 });
```

### `automations.getPrompt(id)` / `automations.setPrompt(id, prompt)`

Читает или заменяет промпт автоматизации без выполнения полного обновления.
