<Callout type="warn">
SDK находится на стадии раннего альфа-тестирования — он не предназначен для использования в продакшене и может быть удалён в будущем. Оставайтесь на последней версии (`@rox_sh/sdk@latest`), пока мы дорабатываем его.
</Callout>

TypeScript SDK для Rox позволяет создавать и управлять вашими рабочими пространствами, задачами, проектами и автоматизациями из кода — всё, что вы можете сделать в панели управления, могут делать и ваши скрипты и сервисы.

Вот несколько вариантов использования:

- **Массовое управление задачами** — автоматизируйте создание, сортировку или очистку задач в одном или многих проектах с помощью скриптов.
- **Интеграции** — синхронизируйте задачи с вашим CRM, трекером задач или внутренними инструментами и обратно.
- **Подготовка окружений** — разворачивайте рабочие пространства на машине разработчика, когда открывается issue, рецензируется PR или поступает обращение клиента.
- **Агенты по запросу** — запускайте регулярную автоматизацию вне расписания при срабатывании вебхука.

Всё полностью типизировано и работает в Node, Bun и Deno. Сборка также работает в браузерах, но **не передавайте ключи `sk_live_…` во фронтенд** — они долгоживущие и дают полный доступ к организации. Из браузера обращайтесь к тонкому серверному прокси, который хранит ключ.

Мы ведём активную разработку — SDK следует за API по мере его роста, поэтому **оставайтесь на последней версии**, а не фиксируйте конкретную. Добавляются новые методы; существующие не меняют свою форму без повышения версии и записи в журнале изменений.

## Получение API-ключа

1. Откройте [десктопное приложение Rox](https://rox.one/download) → **Settings → API Keys** → **Create**. Ключ начинается с `sk_live_…` — скопируйте его сейчас (вы больше не сможете его увидеть).
2. Запишите свой **идентификатор организации** — найдите его в переключателе организаций внутри десктопного приложения.

```bash
export ROX_API_KEY=sk_live_…
export ROX_ORGANIZATION_ID=…
```

## Создание клиента

```ts
import Rox from '@rox_sh/sdk';

const client = new Rox();
// reads ROX_API_KEY and ROX_ORGANIZATION_ID from env
```

На этом вся настройка завершена. Дальше каждый ресурс работает одинаково:

```ts
// Create a task
const task = await client.tasks.create({
  title: 'Wire up auth',
  priority: 'high',
});

// Update it
await client.tasks.update({ id: task.id, statusId: '<uuid>' });

// Look one up by id or slug
const got = await client.tasks.retrieve('SUPER-172'); // returns null if missing

// Find tasks with rich filters
const urgent = await client.tasks.list({
  priority: 'urgent',
  search: 'auth',
  creatorMe: true,
});

// Inspect what else you have
const projects = await client.projects.list();
const hosts = await client.hosts.list();
const automations = await client.automations.list();

// Trigger an automation off-schedule
await client.automations.run('<automation-id>');

// Create a workspace on a developer's machine
await client.workspaces.create({
  hostId: '<machine-id>',
  projectId: '<project-id>',
  name: 'fix-auth-bug',
  branch: 'fix/auth',
});
```

Смотрите [справочник](/docs/sdk/reference), чтобы узнать о каждом методе, параметре и возвращаемом типе.

## Конфигурация

Вы можете настроить клиент явно вместо переменных окружения (или вместе с ними):

```ts
const client = new Rox({
  apiKey: 'sk_live_…',
  organizationId: '…',
  timeout: 60_000,            // default: 60s per request
  maxRetries: 2,              // default: 2 retries with exponential backoff
  logLevel: 'warn',           // 'off' | 'error' | 'warn' | 'info' | 'debug'
});
```

Клиент автоматически повторяет запросы при сетевых ошибках и ответах 429/5xx, поэтому временные сбои обычно устраняются сами собой.

## Обработка ошибок

SDK выбрасывает типизированные ошибки для любого неуспешного ответа, поэтому вы можете перехватывать именно то, что вас действительно интересует:

```ts
import Rox, { APIError, NotFoundError, RateLimitError } from '@rox_sh/sdk';

const client = new Rox();

try {
  await client.tasks.create({ title: '' });
} catch (err) {
  if (err instanceof RateLimitError) {
    // 429 — already retried up to maxRetries; back off further if you keep hitting it
  } else if (err instanceof NotFoundError) {
    // 404
  } else if (err instanceof APIError) {
    // anything else — err.status, err.headers, err.error (the parsed body)
    console.error(`Rox returned ${err.status}:`, err.error);
  }
}
```
