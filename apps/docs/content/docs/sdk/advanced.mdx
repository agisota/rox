Отдельные методы SDK просты. Самое интересное — то, во что вы их объединяете. Это сквозные рецепты, которые вы можете скопировать как отправные точки.

## Вебхук Sentry → агент по багу

Когда Sentry вызывает оповещение, заводите задачу и запускайте агента для расследования — всё в два вызова SDK. (В продакшене проверяйте заголовок с подписью Sentry, прежде чем доверять любому payload — эта часть здесь опущена для ясности.)

```ts title="sentry-webhook.ts"
import { Hono } from 'hono';
import Rox from '@rox_sh/sdk';

const app = new Hono();
const client = new Rox();
// reads ROX_API_KEY + ROX_ORGANIZATION_ID from env

const TARGET_HOST_ID = process.env.ROX_TARGET_HOST_ID!;  // see hosts.list()
const PROJECT_ID = process.env.ROX_PROJECT_ID!;          // see projects.list()

app.post('/webhooks/sentry', async (c) => {
  const event = await c.req.json();
  const { title, web_url: sentryUrl, culprit, short_id } = event.data.issue;
  const slug = `sentry-${short_id.toLowerCase()}`;

  // 1. File a task so the bug shows up in the dashboard
  const task = await client.tasks.create({
    title: `[Sentry] ${title}`,
    description: `Triggered by ${culprit}\n\nSentry: ${sentryUrl}`,
    priority: 'high',
    labels: ['sentry', 'bug'],
  });

  // 2. Spin up a worktree on a developer's machine and dispatch an agent
  //    inside it. The SDK creates the worktree, then runs the agent with
  //    your prompt — you don't manage the dispatch loop.
  const workspace = await client.workspaces.create({
    hostId: TARGET_HOST_ID,
    projectId: PROJECT_ID,
    name: slug,
    branch: slug,
    agents: [
      {
        agent: 'claude',
        prompt: [
          `A Sentry alert just fired:`,
          `  Title: ${title}`,
          `  Culprit: ${culprit}`,
          `  URL: ${sentryUrl}`,
          ``,
          `Reproduce the failure, fix it, push a branch, and update task`,
          `${task.id} with what you found.`,
        ].join('\n'),
      },
    ],
  });

  return c.json({
    ok: true,
    taskId: task.id,
    workspaceId: workspace.id,
    runId: workspace.agentRuns[0]?.runId,
  });
});

export default app;
```

Что произошло, от начала до конца:

- Задача находится в облаке и появляется в панели управления у всех.
- Рабочее пространство — это реальный `git worktree` на целевой машине, готовый к тому, чтобы человек подхватил работу, если агент застрянет.
- Агент работает внутри нового рабочего пространства, с приведённым выше промптом и доступом к кодовой базе + всем MCP-серверам, настроенным на хосте.

Вы можете запустить несколько агентов параллельно, добавив больше элементов в `agents`:

```ts
agents: [
  { agent: 'claude', prompt: 'Reproduce the bug and write a failing test.' },
  { agent: 'claude', prompt: 'Investigate related Sentry issues from the last 7 days.' },
],
```

## Разгрести бэклог P0 за ночь

Подхватите каждую срочную задачу в очереди и запустите агента на каждой из них параллельно. Утром вы получаете ветку и комментарий по каждой задаче.

```ts title="burn-p0s.ts"
import Rox from '@rox_sh/sdk';

const client = new Rox();
const HOST_ID = process.env.ROX_HOST_ID!;
const PROJECT_ID = process.env.ROX_PROJECT_ID!; // see projects.list()

const p0s = await client.tasks.list({ priority: 'urgent', limit: 50 });

const dispatched = await Promise.all(
  p0s.map((task) =>
    client.workspaces.create({
      hostId: HOST_ID,
      projectId: PROJECT_ID,
      name: task.slug,
      branch: task.slug,
      agents: [
        {
          agent: 'claude',
          prompt: [
            `Pick up task ${task.slug}: "${task.title}".`,
            ``,
            task.description ?? '(no description — read recent changes for context)',
            ``,
            `Investigate, make the fix, run the relevant tests, push the branch, and`,
            `update task ${task.id} with a one-paragraph summary of what changed and why.`,
            `If you can't make confident progress in ~30 minutes, leave a comment with`,
            `where you got stuck and stop.`,
          ].join('\n'),
        },
      ],
    }),
  ),
);

console.log(`dispatched ${dispatched.length} agents across the P0 backlog`);
```

Запустите это из ежедневного cron на мощном хосте, и вы проснётесь с веткой на каждый баг. Те, что агент смог исправить, имеют готовую к PR ветку; те, что не смог, — комментарий с объяснением причины, чтобы человек продолжил с того места, где агент остановился.

## Прокатить рефакторинг по каждому репозиторию

Переименование внутреннего API, обновление зависимости, развёртывание правила линтинга — всё, что должно одинаково попасть в десятки репозиториев, — это задача с одним циклом и веерным распределением:

```ts title="sweep-rename-getUser.ts"
import Rox from '@rox_sh/sdk';

const client = new Rox();
const HOST_ID = process.env.ROX_HOST_ID!;

const projects = await client.projects.list();

const wave = await Promise.all(
  projects.map((p) =>
    client.workspaces.create({
      hostId: HOST_ID,
      projectId: p.id,
      name: `rename-getUser-${p.slug}`,
      branch: 'refactor/rename-getUser',
      agents: [
        {
          agent: 'claude',
          prompt: [
            `Rename every caller of \`getUser()\` to \`fetchUser()\` across this repo.`,
            ``,
            `1. Find call sites with grep.`,
            `2. Update each one and re-run the test suite.`,
            `3. If green, push the branch and open a draft PR titled`,
            `   "refactor: rename getUser → fetchUser".`,
            `4. If a repo doesn't actually use getUser, exit cleanly with a note.`,
          ].join('\n'),
        },
      ],
    }),
  ),
);

const totalAgents = wave.reduce((n, ws) => n + ws.agentRuns.length, 0);
console.log(`fanned out ${totalAgents} agents across ${projects.length} repos`);
```

Тот же цикл работает для «обновить TypeScript до 5.7», «заменить Sentry SDK на новый», «добавить скрипт `lint:strict` в каждый package.json». Всё, что иначе вы бы отдали в длинный тред Slack.

## Запуск агента по запросу из ваших инструментов

Если вы уже создали регулярную автоматизацию в панели управления, SDK может запустить её вне расписания откуда угодно — slash-команда Slack, CI-задача, внутренняя админ-панель:

```ts
const run = await client.automations.run('<automation-id>');
console.log(`dispatched run ${run.id} to host ${run.hostId}`);
```

Закреплённый хост автоматизации должен быть в сети и туннелировать. SDK выдаст ошибку `503 Host not connected`, если это не так, чтобы вы могли повторить попытку позже или вызвать кого-нибудь.
