И Rox, и Warp помогают разработчикам работать с AI вокруг командной строки, но они решают разные проблемы. Warp — это современный эмулятор терминала со встроенными AI-агентами. Rox — это local-first рабочее пространство для запуска множества агентов кода параллельно, каждого в своём изолированном Git worktree.

## Что такое Rox?

Rox — это local-first десктопное рабочее пространство для AI-агентов кода. Ты поднимаешь 10+ агентов на отдельных задачах одновременно, каждого в своём Git worktree с выделенной веткой и рабочей директорией. Вокруг этого ядра Rox добавляет встроенный редактор diff/файлов, панель чата, внутренний браузер для документации и dev-серверов, управление портами и инструменты MCP. Rox не поставляет обязательного слоя модели — ты можешь принести Claude Code, Codex, Aider, OpenCode или других агентов, и ты можешь делать ревью прямо в Rox или перейти в редактор по своему выбору. Открытый исходный код под Elastic License 2.0 (ELv2).

---

## Что такое Warp?

Warp — это эмулятор терминала на Rust с GPU-ускорением, который вырос в Agentic Development Environment. Warp 2.0 добавил встроенных AI-агентов, редактор кода с ревью diff и Warp Drive для обмена командными знаниями. Agent Mode принимает промпты на естественном языке, выполняет многошаговые воркфлоу и самокорректируется при ошибках. Agents 3.0 позволяет агентам взаимодействовать с REPL, отладчиками и интерактивными CLI-инструментами. Доступен на macOS, Windows и Linux.

## Ключевые отличия

### Встроенные агенты против «принеси своих»

Warp поставляет собственных агентов — вводишь промпт, получаешь результат, никаких внешних инструментов ставить не нужно. Rox по умолчанию больше про «принеси своих»: ты устанавливаешь Claude Code, Codex, Aider или другого агента, и Rox запускает их параллельно. Warp удобнее из коробки; Rox даёт тебе полную свободу в выборе агента, модели и провайдера, при этом добавляя вокруг них собственный чат и инструменты рабочего пространства.

### Параллелизм и изоляция

Оба инструмента поддерживают несколько агентов, но модель изоляции различается. Rox даёт каждому агенту выделенный Git worktree — отдельная ветка, отдельная рабочая директория — так что агенты не могут изменять файлы друг друга. Warp запускает агентов в панелях терминала прямо в твоей файловой системе. Для нескольких агентов на несвязанных задачах это работает. Для десяти агентов на одной кодовой базе изоляция через worktree предотвращает конфликты.

### Качество терминала против фокуса на оркестрации

Warp — отполированный терминал: GPU-ускорение, вывод на основе блоков и палитра команд. Rox включает терминалы, потому что агентам они нужны, но продукт всё больше ближе к редактору кода для AI-агентов, чем к классическому терминалу на каждый день. Для SSH и разовых команд ты всё равно будешь использовать Warp, iTerm2 или Kitty.

### Командная совместная работа

У Warp есть командные функции через Warp Drive: общие команды, ноутбуки, переменные окружения плюс интеграции со Slack/Linear/GitHub. У Rox нет функций командной совместной работы.

## Цена

Rox предлагает бесплатный тариф и Pro за $20/место/месяц. Ты также платишь за API-ключи, которые потребляют выбранные тобой агенты.

Терминальные функции Warp бесплатны. AI-функции требуют плана Build ($20/месяц, 1 500 кредитов) или BYOK. Business стоит $50/польз/месяц с SSO и нулевым хранением данных.

## Что выбрать тебе?

**Выбирай Warp, если ты:**
- Хочешь all-in-one терминал со встроенными AI-агентами, редактированием кода и командными инструментами
- Ценишь полировку терминала — GPU-рендеринг, вывод на основе блоков, палитру команд
- Нуждаешься в кросс-платформенной поддержке (macOS, Windows, Linux)
- Предпочитаешь встроенных агентов, которые работают без настройки внешних инструментов

**Выбирай Rox, если ты:**
- Запускаешь множество агентов кода параллельно и нуждаешься в изоляции на уровне Git между ними
- Хочешь полный контроль над тем, какого агента и модель ты используешь, без привязки
- Требуешь инструментов с открытым исходным кодом и локальными Git worktree
- Уже имеешь терминал, который тебе нравится, и нуждаешься в слое оркестрации поверх

Эти инструменты дополняют друг друга. Используй Warp как свой терминал на каждый день, а Rox — для оркестрации параллельных агентов по worktree.

## Часто задаваемые вопросы

### Заменяет ли Rox собой Warp?

Нет. Rox — не твой обычный терминал на каждый день. Теперь он включает чат, браузер и ревью diff/файлов для агентских воркфлоу, но он по-прежнему специализирован вокруг оркестрации агентов кода параллельно. Ты можешь использовать оба: Warp для повседневной работы в терминале, Rox для параллельных агентских воркфлоу.

### Изолирует ли Warp агентов с помощью Git worktree?

Нет. Агенты Warp оперируют прямо в твоей файловой системе. Если два агента одновременно изменяют один и тот же файл, могут возникнуть конфликты. Rox предотвращает это, давая каждому агенту свой worktree, ветку и директорию.

### Можно ли запускать Claude Code или Codex в обоих инструментах?

Да. Warp запускает их в панелях терминала вместе со своими встроенными агентами. Rox запускает их в изолированных Git worktree с параллельным выполнением как основным воркфлоу. Для одного-двух агентов подойдёт любой. Для десяти агентов на десяти задачах с изоляцией веток Rox создан специально под это.
