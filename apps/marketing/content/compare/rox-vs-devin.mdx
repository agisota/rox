Devin — это облачный AI-инженер ПО, который работает автономно в удалённой виртуальной машине. Rox — это local-first десктопное рабочее пространство, которое оркестрирует агентов кода параллельно по Git worktree. Они представляют противоположные концы спектра разработки с AI: Devin выгружает работу в облако, тогда как Rox держит репозиторий, поток ревью и среду исполнения агентов привязанными к твоей машине.

## Что такое Rox?

Rox — это local-first десктопное рабочее пространство для AI-агентов кода. Оно запускает Claude Code, Codex, OpenCode, Aider, Copilot, Cursor Agent, Gemini CLI, Rox Chat и другие агентские воркфлоу внутри изолированных Git worktree с постоянными сессиями терминала. Вокруг этого ядра добавлены встроенный редактор diff/файлов, панель чата, внутренний браузер для документации и dev-серверов, управление портами и инструменты MCP. Ты можешь делать ревью прямо в Rox или перейти в VS Code, Cursor, Windsurf, JetBrains или Xcode. Открытый исходный код под Elastic License 2.0 (ELv2).

---

## Что такое Devin?

Devin — это AI-инженер ПО от Cognition: облачный автономный агент, который работает в собственной виртуальной машине с браузером, редактором кода, терминалом и планировщиком. Ты назначаешь задачи через чат или Slack, и Devin работает самостоятельно: читает документацию, пишет код, отлаживает, запускает тесты и создаёт pull request'ы. У него есть событийно-управляемые воркфлоу (триггеры из Linear, Slack и т.д.) и система ревью, которая аннотирует PR перед ревью человеком.

## Ключевые отличия

### Локально против облака

Поставляемый воркфлоу Rox сегодня local-first. Твой репозиторий, worktree, поток ревью, чат и превью в браузере остаются на твоей машине, а исходящий трафик зависит от конкретных агентов или провайдеров, которых ты выбираешь. Devin работает целиком в облаке Cognition — твой код клонируется в удалённые ВМ, где оперирует Devin. Это фундаментальное архитектурное различие, и оно определяет большинство компромиссов ниже.

### Контроль против автономии

Rox даёт тебе прямой доступ к каждой сессии агента. Ты можешь следить за терминалом, инспектировать diff, открыть превью в браузере, общаться в чате в том же рабочем пространстве, прервать или перенаправить работу посреди задачи. Devin стремится к полной автономии — ты назначаешь задачу и возвращаешься позже за PR. Это делает Devin более «без рук», но менее контролируемым, когда он идёт в неправильном направлении.

### Гибкость в выборе агента

Devin — это единственный проприетарный агент: ты используешь AI от Cognition или ничего. Rox не зависит от агента: запускай Claude Code для сложных рефакторингов, Codex для чётко очерченных задач, Aider для итеративной работы. Когда выходит агент получше, используй его в Rox сразу же. С Devin ты ждёшь, пока Cognition улучшит свою модель.

### Механизм изоляции

Rox изолирует агентов с помощью Git worktree — лёгких, быстрых и встроенных в сам git. Devin изолирует каждую сессию в полноценной облачной ВМ со своей ОС, браузером и тулчейном. ВМ дают более сильную изоляцию и более полное соответствие окружения, но при гораздо более высокой стоимости и задержке.

### Цена

Rox предлагает бесплатный тариф и Pro за $20/место/месяц плюс расходы на API твоих агентов. Devin стоит $500/место/месяц по плану Teams. Разница в цене существенная — Rox плюс оплата API при тяжёлом использовании агентов обычно стоит долю от месячной платы Devin.

## Что выбрать тебе?

**Выбирай Devin, если ты:**
- Хочешь полностью автономный AI, который работает в фоне с минимальным надзором
- Нуждаешься в событийно-управляемых воркфлоу (автоответ на тикеты Linear, сообщения Slack)
- Предпочитаешь облачное выполнение, где ты не управляешь локальными вычислениями
- Имеешь бюджет на $500/место/месяц и ROI это оправдывает

**Выбирай Rox, если ты:**
- Хочешь прямой контроль над каждым агентом — видеть, что они делают, перенаправлять в реальном времени
- Нуждаешься в локальном выполнении и прямом контроле над каждым агентом
- Хочешь гибкость в выборе агента (Claude Code, Codex, Aider, OpenCode — твой выбор под каждую задачу)
- Предпочитаешь экономичность локального выполнения с прямой оплатой API
- Нуждаешься в инструменте с открытым исходным кодом, который можно инспектировать и изменять

## Часто задаваемые вопросы

### Могут ли Devin и Rox работать вместе?

Они обслуживают разные воркфлоу. Devin — это облачный автономный инженер. Rox — это локальный оркестратор агентов. Ты можешь использовать Devin для полностью автономных фоновых задач, а Rox — для интерактивной параллельной работы, где тебе нужно больше контроля.

### Стоит ли Devin своей ценовой премии?

$500/место/месяц у Devin покупают полную автономию: назначаешь задачи через Slack, получаешь PR обратно. Если твоему воркфлоу выгодно делегирование «без рук» и у тебя есть бюджет, это может быть ценно. Если ты предпочитаешь прямой контроль и экономичность, Rox за $20/место/месяц плюс расходы на API даёт больше пропускной способности на доллар.

### Open source ли Rox?

Нет. Rox — с открытым исходным кодом на GitHub под Elastic License 2.0 (ELv2). Devin — с закрытым исходным кодом от Cognition.

### Что лучше для чувствительных к безопасности кодовых баз?

Rox — более безопасный выбор по умолчанию, если local-first выполнение является жёстким требованием. Devin запускает код в облачной инфраструктуре Cognition. Для регулируемых отраслей, государственной работы или чувствительной интеллектуальной собственности сохранение репозитория и цикла ревью на собственной машине часто оказывается решающим фактором.
