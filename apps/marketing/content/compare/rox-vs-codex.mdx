Codex CLI — это терминальный агент для кода от OpenAI, но теперь он находится внутри более широкого продукта Codex, который также охватывает расширения для IDE, десктопное приложение и облачные воркфлоу. Rox — это local-first десктопное рабочее пространство, которое умеет запускать несколько инстансов Codex, других агентов и Rox Chat параллельно в изолированных Git worktree. Продукты пересекаются сильнее, чем раньше, но они по-прежнему решают разные задачи: Codex — это кодинг-набор, а Rox — слой оркестрации и ревью вокруг множества агентов.

## Что такое Rox?

Rox — это local-first десктопное рабочее пространство для AI-агентов кода. Оно запускает Codex, Claude Code, OpenCode, Aider, Copilot, Cursor Agent, Gemini CLI, Rox Chat и другие агентские воркфлоу внутри изолированных Git worktree с постоянными сессиями терминала. Вокруг этого ядра добавлены встроенный редактор diff/файлов, панель чата, внутренний браузер для документации и dev-серверов, управление портами и инструменты MCP. Ты можешь делать ревью прямо в Rox или перейти в VS Code, Cursor, Windsurf, JetBrains или Xcode. Открытый исходный код под Elastic License 2.0 (ELv2).

---

## Что такое Codex CLI?

Codex CLI — это open-source локальный терминальный клиент Codex от OpenAI. Он работает на твоей машине, понимает твою кодовую базу через чтение файлов и выполнение команд и может писать код, чинить баги и рефакторить по файлам. Теперь OpenAI подаёт Codex как более широкий кодинг-набор, охватывающий CLI, расширения для IDE, десктопное приложение и делегирование задач в облако. Приложение Codex здесь особенно актуально, потому что OpenAI теперь позиционирует его как способ запускать несколько агентов Codex параллельно по проектам со встроенной поддержкой worktree, скиллами, автоматизациями и функциональностью Git.

## Ключевые отличия

### Оркестратор против агентского набора

Codex — это агентский набор: он читает код, общается с моделями OpenAI и вносит изменения через локальные и облачные интерфейсы. Rox — это рабочее пространство для оркестрации вокруг агентов: оно запускает множество сессий Codex параллельно, каждую в своём изолированном окружении, а затем даёт тебе чат, ревью, превью в браузере и воркфлоу с подключением через MCP вокруг них. Запусти Codex внутри Rox, чтобы получить автоматическую изоляцию через worktree и параллельное выполнение без ручной настройки.

### Локально, в приложении и в облаке против локальной оркестрации

Codex теперь охватывает несколько интерфейсов: CLI, расширение для IDE, десктопное приложение и делегирование в облако. Rox сегодня local-first: worktree, терминалы, интерфейс чата, вкладки браузера и ревью diff/файлов — всё живёт на твоей машине, а трафик к провайдеру зависит от того, каких агентов ты запускаешь. Это делает связь яснее: Codex предоставляет кодинг-интерфейсы, а Rox предоставляет локальный слой оркестрации вокруг множества задач.

### Один продуктовый стек против стека из разных агентов

С одним только Codex — будь то из CLI или приложения — ты остаёшься внутри продуктовой модели Codex от OpenAI. С Rox ты можешь назначать задачи нескольким инстансам Codex одновременно — или смешивать Codex с Claude Code, Aider, OpenCode или Rox Chat — каждый на своей ветке со своим worktree.

### Гибкость в выборе агента

Codex CLI работает только с моделями OpenAI. Rox всё равно, каким агентом ты пользуешься — запускай Codex для задач, где модели OpenAI блистают, Claude Code для сложной архитектурной работы и Aider для итеративных изменений. Комбинируй под каждую задачу в зависимости от того, что работает лучше.

### Сохранность сессий

Codex даёт тебе агентские интерфейсы. Rox добавляет операционный слой вокруг них: worktree на уровне задач, долгоживущие сессии, превью локальных приложений в браузере, встроенное ревью diff/файлов и координацию с учётом MCP. Это важно, когда ты управляешь множеством задач на одной кодовой базе, а не одним промптом в одном интерфейсе.

## Цена

Rox предлагает бесплатный тариф и Pro за $20/место/месяц. Цена Codex теперь зависит от того, каким интерфейсом ты пользуешься: OpenAI открывает Codex через тарифы ChatGPT, локальные и облачные разрешения и воркфлоу на основе API. Важное практическое различие всё то же: Codex даёт тебе агентские интерфейсы OpenAI, а Rox даёт слой оркестрации для запуска множества локальных задач параллельно.

## Что выбрать тебе?

**Выбирай один только Codex CLI, если ты:**
- Работаешь над одной задачей за раз и предпочитаешь простой терминальный воркфлоу
- Хочешь семейство моделей Codex от OpenAI и его продуктовый стек для задач кодинга
- Предпочитаешь режим Full Auto для автономного выполнения в изолированном окружении
- Нуждаешься в облачном Codex для тяжёлых задач, которым выгодны удалённые вычисления

**Выбирай Rox + Codex CLI, если ты:**
- Хочешь запускать 5-10+ инстансов Codex параллельно по разным задачам
- Нуждаешься в автоматической изоляции через Git worktree, чтобы агенты никогда не конфликтовали
- Хочешь сессии, которые переживают сбои и перезапуски приложения
- Планируешь смешивать Codex с другими агентами (Claude Code, Aider, OpenCode) под каждую задачу
- Работаешь над большими кодовыми базами, где параллельное выполнение экономит часы

Rox и Codex CLI естественно работают вместе. Каждая задача Rox запускает Codex в своём worktree — ты получаешь способность Codex писать код с параллелизмом и изоляцией Rox.

## Часто задаваемые вопросы

### Заменяет ли Rox собой Codex?

Не совсем. У Rox теперь есть встроенный чат, инструменты MCP, превью в браузере и ревью файлов, но его главная задача по-прежнему — координировать агентов и worktree. Codex — это агентский набор для кода. Rox — это слой, который позволяет безопасно запускать множество сессий Codex или смешивать Codex с другими агентами на одном репозитории.

### Можно ли запускать Codex в режиме Full Auto внутри Rox?

Да. Каждая задача Rox запускает Codex в своём изолированном worktree. Ты можешь безопасно использовать любой режим Codex — Suggest, Auto Edit или Full Auto, — потому что изоляция через worktree означает, что автономный агент не может затронуть твою основную рабочую директорию.

### Чем приложение Codex отличается от Rox?

Приложение Codex даёт тебе нативный для OpenAI GUI для самого Codex со встроенными worktree, скиллами, автоматизациями и поддержкой Git. Rox шире в другом направлении: он не привязан к Codex и может оркестрировать множество разных агентов, при этом давая тебе чат, превью в браузере и встроенное ревью diff/файлов. Если тебе нужно приложение конкретно под Codex, используй приложение Codex. Если тебе нужен control plane без привязки к агенту, используй Rox.

### Оба ли инструмента open source?

Нет. Codex CLI — open source под Apache 2.0. Rox — с открытым исходным кодом на GitHub под Elastic License 2.0 (ELv2).
