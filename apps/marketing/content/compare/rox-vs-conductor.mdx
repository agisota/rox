## Что такое Rox?

Rox — это local-first десктопное рабочее пространство для AI-агентов кода. Оно запускает Claude Code, Codex, OpenCode, Aider, Copilot, Cursor Agent, Gemini CLI, Rox Chat и другие агентские воркфлоу внутри изолированных Git worktree с постоянными сессиями терминала. Вокруг этого ядра добавлены встроенный редактор diff/файлов, панель чата, внутренний браузер для документации и dev-серверов, управление портами и инструменты MCP. Ты можешь делать ревью прямо в Rox или перейти в VS Code, Cursor, Windsurf, JetBrains или Xcode. Открытый исходный код под Elastic License 2.0 (ELv2).

---

## Что такое Conductor?

Conductor — это приложение для macOS от Melty Labs для параллельного запуска Claude Code и Codex. Его главная страница позиционирует его как Mac-приложение, которое клонирует твой репозиторий, создаёт изолированные рабочие пространства и позволяет делать ревью и мержить изменения от нескольких агентов. Документация делает акцент на руководствах по воркфлоу — настройке issue-to-PR, конфигурации провайдеров, поддержке Codex и открытии рабочих пространств в Cursor или VS Code.

## Ключевые отличия

### Гибкость в выборе агента

Rox намеренно широк — любой процесс, который запускается в шелле, может вписаться в его воркфлоу, и текущий продукт также добавляет вокруг этих агентов чат, превью в браузере и инструменты ревью. Conductor сейчас поддерживает Claude Code и Codex. Если это два твоих основных агента, такая сфокусированность может быть преимуществом. Если ты хочешь подмешать OpenCode, Aider, кастомные скрипты или то, что выйдет в следующем месяце, Rox даёт больше пространства для адаптации.

### Воркфлоу Mac-приложения против терминала без привязки к агенту

Conductor создан специально вокруг своего воркфлоу Mac-приложения: клонируй репозиторий, разверни агентов Claude Code или Codex, сделай ревью работы, затем смержи. Rox шире и меньше привязан к одной паре агентов. Теперь у него есть собственные чат, ревью diff/файлов и интерфейсы браузера, но он по-прежнему держит слой оркестрации независимым от лежащих в основе агентов. Это делает Conductor проще для понимания, если тебе нужно одно отполированное приложение под узкий набор агентов, тогда как Rox лучше, если тебе нужна гибкость.

### Локальные рабочие пространства против worktree существующего репозитория

На текущей главной странице Conductor сказано, что он клонирует твой репозиторий и работает целиком на твоём Mac, затем создаёт изолированное рабочее пространство для каждого агента. Rox обычно стартует из репозитория, который у тебя уже открыт, и создаёт дополнительные Git worktree вокруг этого существующего чекаута. Оба используют worktree; разница в том, вокруг какого локального воркфлоу они обёрнуты.

### Интеграция воркфлоу

Документация Conductor опирается на руководства по воркфлоу: issue-to-PR, настройка провайдеров, миграция с Cursor и открытие рабочих пространств в Cursor или VS Code. Rox интегрируется с большим числом редакторов и более широкими комбинациями CLI-агентов, и теперь он включает больше собственной функциональности внутри самого приложения, но при этом оставляет на тебя больше окружающего процесса, чем узко сфокусированный продукт.

## Что выбрать тебе?

**Выбирай Rox, если ты:**

- Используешь несколько CLI-агентов или хочешь свободу мгновенно осваивать новые
- Хочешь, чтобы оркестрация оставалась отдельной от любого вендора агентов или приложения
- Предпочитаешь делать ревью кода в своём стеке редакторов (JetBrains, Xcode, VS Code, Cursor)
- Нуждаешься в более широкой модели «принеси своего агента»

**Выбирай Conductor, если ты:**

- В основном используешь Claude Code и Codex и хочешь сфокусированное Mac-приложение под них
- Хочешь отполированный воркфлоу в стиле issue-to-PR вокруг этих двух агентов
- Предпочитаешь дашборд рабочих пространств Conductor и документированную передачу в Cursor/VS Code

**Вердикт:** Conductor сильнее всего, если твой мир уже вращается вокруг Claude Code и Codex на macOS. Rox подходит лучше, если ты ожидаешь, что твой набор агентов будет меняться, хочешь, чтобы слой оркестрации оставался независимым от редактора, или ценишь широту больше, чем узко очерченный воркфлоу приложения.

## Часто задаваемые вопросы

### Только ли для Mac предназначен Conductor?

Conductor сейчас продвигает себя как Mac-приложение. Язык его главной страницы и поток загрузки ориентированы на Mac, тогда как сравнительная история Rox — про оркестрацию CLI-агентов в более широком смысле.

### Может ли Conductor запускать агентов помимо Claude Code?

Текущее публичное позиционирование Conductor — это Claude Code и Codex. Если тебе нужны OpenCode, Aider или кастомные CLI-инструменты в одном оркестраторе, Rox — более гибкий выбор.

### Оба ли инструмента используют Git worktree?

Да. Оба создают изолированные Git worktree для каждого агента, давая каждой задаче свою ветку и рабочую директорию. Лежащий в основе механизм изоляции один и тот же.
