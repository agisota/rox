Rox теперь можно использовать из куда большего числа мест, чем десктопное приложение: проработанный CLI и TypeScript SDK, Slack-агент, который сам поднимает рабочие пространства, и мобильное приложение, выходящее на внутреннее ревью.

## Автоматизации

**Автоматизации — это cron-задачи для сессий агентов.** Выбери проект, напиши промпт, задай расписание — и Rox запускает прогон агента в свежем рабочем пространстве каждый раз, когда оно срабатывает. Результат — живое рабочее пространство, которое ты можешь открыть, просмотреть и продолжить интерактивно.

Типичные применения:

- **Ночные стендапы** — суммируй PR'ы, issue и активность за последние 24 часа
- **Release notes** — черновик из смерженных коммитов каждую неделю
- **Проверки безопасности и зависимостей** — сканируй уязвимые пакеты или устаревшие API каждое утро
- **Долгие рефакторинги** — «обнови TypeScript до 5.7 в следующем не мигрированном пакете» на ежедневной основе

Этот набор выводит автоматизации в полноценный воркфлоу:

- **История версий промптов** <PRBadge url="https://github.com/agisota/set/pull/3996" /> — каждая правка промпта автоматизации снимается в снапшот. Просматривай предыдущие версии и восстанавливай любую из них, так что случайное изменение больше не дверь в одну сторону.
- **Переработанная вкладка автоматизаций** <PRBadge url="https://github.com/agisota/set/pull/4225" /> — новая компоновка, которая упрощает быстрый просмотр очереди прогонов.
- **Фильтр «Мои» / «Команда»** <PRBadge url="https://github.com/agisota/set/pull/4403" /> — автоматизации теперь принадлежат команде. Фильтруй представление прогонов между **«Мои»** и **«Команда»**, чтобы видеть только то, что запустил ты, или всё, что запустила твоя команда.

### Управляй ими из CLI

Всё, что ты можешь сделать в интерфейсе, теперь скриптуется. Новое в этом наборе: `prompt get` / `prompt set` <PRBadge url="https://github.com/agisota/set/pull/3966" /> и фильтр `--name` при выводе списка <PRBadge url="https://github.com/agisota/set/pull/3959" />.

```bash
# create a recurring run from a prompt file
rox automations create \
  --name "Nightly audit" \
  --rrule "FREQ=DAILY;BYHOUR=3;BYMINUTE=0" \
  --project <projectId> \
  --agent claude \
  --prompt-file prompt.md

# manage the queue
rox automations list --name "Nightly"
rox automations pause <id>
rox automations resume <id>

# iterate on the prompt (backed by the new version history)
rox automations prompt get <id> > prompt.md
$EDITOR prompt.md
rox automations prompt set <id> --file prompt.md

# fire one off-schedule (also works from the SDK / Slack bot / MCP)
rox automations run <id>
```

`rox automations run` также доступен через **TypeScript SDK** и **MCP**, так что ты можешь запускать прогоны из CI-задачи, внутренней админ-панели или slash-команды Slack. Полный обзор смотри в [документации по автоматизациям](https://docs.rox.one/docs/automations).

## Rox CLI

CLI `rox` обзавёлся настоящими командами для проектов и рабочих пространств и сценарием входа между устройствами.

**Новые команды:**

- `rox projects create` <PRBadge url="https://github.com/agisota/set/pull/4355" /> — создай новый проект Rox из своего шелла.
- `rox projects setup` <PRBadge url="https://github.com/agisota/set/pull/4356" /> — свяжи существующий локальный репозиторий с проектом Rox.
- `rox workspaces open` <PRBadge url="https://github.com/agisota/set/pull/4258" /> — открой рабочее пространство прямо из терминала.
- `rox auth login` <PRBadge url="https://github.com/agisota/set/pull/3965" /> — теперь использует OAuth device code + PKCE, так что работает на безголовых серверах и между устройствами.

**Как это настроить:**

```bash
# install
curl -fsSL https://rox.one/cli/install.sh | sh
# or: brew install agisota/tap/rox
# or: it's already bundled with the desktop app at ~/.rox/bin/rox

# sign in (opens a browser, or prints a code for a second device)
rox auth login

# scaffold a project and open a workspace
rox projects create
rox workspaces open
```

Сопутствующий **TypeScript SDK** <PRBadge url="https://github.com/agisota/set/pull/3937" /> (`@rox_sh/sdk`) зеркалит CLI один в один — те же процедуры, те же формы — так что всё, что ты можешь сделать из шелла, можно сделать из скрипта, CI-задачи или внутреннего инструмента.

```bash
npm install @rox_sh/sdk
```

```ts
import Rox from '@rox_sh/sdk';
const client = new Rox(); // reads ROX_API_KEY / ROX_ORGANIZATION_ID
await client.automations.run('<automation-id>');
```

`rox start` и `rox update` на Linux теперь исправлены <PRBadge url="https://github.com/agisota/set/pull/4635" />.

## Slack-бот

![Slack-бот Rox поднимает рабочее пространство и отчитывается прямо в треде](/changelog/slack.png)

`@rox` в Slack теперь настоящий агент, а не заполнитель задач. Упомяни его в канале или напиши ему в личку откуда угодно, и он может:

- **Создавать, обновлять, искать и сортировать задачи** прямо из разговора, в котором находится — подтягивая исполнителей, статусы и приоритеты из настроек Linear/Rox твоей команды.
- **Поднять облачное рабочее пространство и запустить кодирующего агента**, чтобы реально сделать работу. Для запросов на изменение кода (например, «@rox обнови наш Sentry SDK до v9 во всех приложениях») он теперь по умолчанию создаёт рабочее пространство и агента, а не просто заводит тикет <PRBadge url="https://github.com/agisota/set/pull/4660" />.
- **Читать историю треда или канала** самостоятельно, чтобы собрать контекст — не нужно копировать предыдущие сообщения.
- **Искать в интернете со ссылками на источники**, когда запросу нужна актуальная информация.
- **Разворачивать ссылки на задачи и рабочие пространства Rox** с богатыми превью, когда ими делятся в канале.

**Как это настроить:**

1. В дашборде перейди в **«Интеграции → Slack»**.
2. Нажми **«Подключить»**, чтобы установить приложение Rox Slack в своё рабочее пространство (OAuth).
3. Упомяни `@rox` в любом канале или открой личку с ботом и опиши, что хочешь сделать.

Если Slack уже был подключён до этого релиза, ничего делать не нужно — проактивное создание рабочих пространств включено по умолчанию.

## Мобильное приложение: внутреннее ревью

Мобильное приложение на **внутреннем ревью**. Основные сценарии — список рабочих пространств, сортировка задач, чат с агентом и живой просмотрщик терминала — подключены к тем же API, что web и десктоп. Мы сейчас тестируем на своих устройствах перед более широкой бетой.

## Также в этом наборе

- **Команды как полноценная сущность** <PRBadge url="https://github.com/agisota/set/pull/4403" /> — назначай участников, рабочие пространства и автоматизации команде. Питает разделение автоматизаций «Мои»/«Команда» выше.
- **Сессии терминала переживают обновления приложения** <PRBadge url="https://github.com/agisota/set/pull/3971" /> — демон PTY передаёт файловые дескрипторы при обновлении, так что запущенные шеллы остаются живыми, когда Rox авто-обновляется.
- **Пресеты веб-терминала** <PRBadge url="https://github.com/agisota/set/pull/4653" /> — запускай свои частые команды одним кликом в веб-терминале, как на десктопе.
