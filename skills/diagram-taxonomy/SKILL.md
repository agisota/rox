# Diagram Taxonomy

Choose the lightest diagram that answers the question:
- ASCII architecture -> boundaries, modules, ownership, blast radius
- sequence -> who calls whom and in what order
- data-flow -> where data moves and transforms
- ERD -> how data is shaped
- state machine -> how status/lifecycle changes over time
- component/screen map -> what the workflow surface looks like
