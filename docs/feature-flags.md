## Lifecycle

- Request: new Agent-Native Team OS capabilities are added to `packages/shared/src/experimental-features`.
- Design: every entry declares owner, type, dependencies, affected surfaces, kill switch, telemetry event, and cleanup trigger.
- Ship: code reads effective state through the desktop settings router or renderer hook instead of hardcoding local booleans.
- Ramp: defaults stay on, but provider-backed features no-op safely as `needs_configuration`.
- Cleanup: when a feature graduates, remove its toggle and dead disabled branch, then archive this row in the release notes.
