● 01 — The primitives

Before the system, the atoms.

A new way of working needs a new vocabulary. Eight primitives replace “agent, tool, prompt.” Hover any atom to read its law. Everything downstream is built from these.

ATOM 01
State
A set of truth conditions about the world at a moment. Either they hold, or they don’t.
ATOM 02
Target State
The verified condition you want true. The system’s gravity — everything aligns toward it.
ATOM 03
Transition
A directional change from one state to the next. The real unit of work — not the agent.
ATOM 04
Event
A minimal occurrence that causes a transition. Enough of the right events open the gate.
ATOM 05
Runtime
The place a transition executes — editor, browser, repo, shell, database. Context isn’t enough; work needs a where.
ATOM 06
Monad
The sufficient set of preconditions a transition needs: context, tools, permissions, criteria, trace. The real bottleneck.
ATOM 07
Trace
The observable path the work left behind. If a transition can’t be observed, it can’t be trusted.
ATOM 08
Validator
A gate that opens only when acceptance criteria pass. Output is not completion — a verified state is.
● 02 — Composition

The task is the difference between two states.

You don’t invent steps. You compute them. The work is exactly Δ = S* − S₀ — the truth conditions that are missing — and each one becomes a transition.

“Do not decompose by agents. Decompose by what must become true.”

● 03 — The shift

Two ways to hold the same problem.

The old pose asks how: which tool, which agent, which prompt. It feels fast and ends in rework. The new pose asks what: which state must change — and arrives.
