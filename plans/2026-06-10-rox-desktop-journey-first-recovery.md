# Rox Desktop 2.0.x — Journey-first recovery plan

Дата: 2026-06-10. Контекст: установленный Rox 2.0.0 говорит с устаревшим прод-API
(деплои Vercel молча падали из-за отсутствия VERCEL_* секретов в CI), отсюда каскад:
вход GitHub/Google не работает, «Организация не найдена», нет команд. Отдельно:
релизный билд ad-hoc подписан → electron-updater на macOS не работает → «Update failed».

## Решения

- **Подпись/нотаризация: ПОКА БЕЗ НЕЁ** (решение Mark, 2026-06-10). Авто-апдейт на macOS
  не работает для неподписанных билдов — обновление вручную через DMG. Тост
  «Update failed» подавить для unsigned-сборок.
- **Переходная совместимость заголовков**: API принимает и `x-rox-organization-id`,
  и legacy `x-superset-organization-id` (старые установленные клиенты).
- **Релиз = полный проход critical journey** на установленном DMG против прода,
  с Peekaboo-скриншот-пруфом каждого шага. Никаких релизов «по коммиту».

## Critical journey (приёмка)

1. Установить опубликованный DMG (текущий целевой: v2.0.1).
2. Вход через GitHub (deep-link rox:// возвращает в приложение, сессия создаётся).
3. Вход через Google (аналогично).
4. Организация видна, «Организация не найдена» отсутствует.
5. Команды: список и настройки команд открываются.
6. Прозрачность («Стеклянные поверхности») реально работает.
7. Нет тоста «Update failed».

## Очередь работ

1. ✅ Журнал миграций прода (0067) — починен, Deploy Database Migrations зелёный.
2. ✅ VERCEL_* секреты в CI — деплой api/web/marketing/docs/admin запущен.
3. CI main: разобрать падения Test + Typecheck (после фикса trpc-моков).
4. Проверить прод-API контракт после деплоя (header, OAuth endpoints) curl'ом.
5. Header-shim в API (оба заголовка).
6. Поставить v2.0.1, пройти journey, зафиксировать блокеры по шагам.
7. Блокер-фиксы строго в порядке journey (вход → орг → команды → UI).
8. Подавить «Update failed» для unsigned-билдов.
9. Прозрачность/vibrancy.
10. Перерелиз и финальная приёмка по чек-листу выше.

## Системные правила (чтобы не повторилось)

- Деплой Production обязан быть зелёным прежде чем резать desktop-тег.
- Переименование клиент-серверных контрактов — только с переходным дуал-чтением на сервере.
